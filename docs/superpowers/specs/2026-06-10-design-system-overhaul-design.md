# 프론트엔드 디자인 전면 개편 — 디자인 시스템 설계

- 작성일: 2026-06-10
- 상태: 설계 승인됨 (구현 계획 전)
- 범위: "대학생 프로젝트" 느낌의 현재 UI를 **업계표준 비즈니스 디자인**으로 개편. 디자인 토큰·컴포넌트 시스템 도입 + 보라색 설비 제거(ISA-101 무채색+상태색) + 로그인~메인 전 화면 적용. 방향 = **C(Industrial Utility / ISA-101)**, 라이트 회색 사이드바, 스틸블루 강조.

---

## 1. 배경 / 디자인 철학
현재(감사 결과): 디자인 시스템 0(ad-hoc Tailwind), 이모지·수제 SVG 아이콘, **보라 설비색**(#a855f7 PITR/제어케이블, #8b5cf6 RTU), 스프레드시트형 밀집 테이블, 밋밋한 로그인/셸, 재사용 컴포넌트 없음.

업계표준 근거:
- **ISA-101 고성능 HMI(IEC 63303)**: 정상 설비는 **무채색(회색)**, 색은 **상태/알람**(빨강·주황·노랑·파랑)에만 — 장식 금지. → 설비를 색으로 칠하지 않는다, 색=정보.
- **엔터프라이즈 B2B SaaS**: 구조적 중립색(차콜/슬레이트/웜그레이) + **절제된 1개 강조색**, 기능적 색. 범주색 5~7개 한정.

채택: **방향 C** — 따뜻한 회색 베이스 + 절제된 스틸블루, 색은 상태에만. 사이드바 라이트 회색.

## 2. 목표 / 비목표

### 목표
1. **디자인 토큰**(CSS 변수 + Tailwind 매핑) — 색·서체·간격·라운드·그림자 단일 소스.
2. **컴포넌트 프리미티브** — Button, Card, Badge, Input/Select, Table, Modal, IconButton, Tabs 등(`components/ui/`). 인라인 Tailwind 난립 제거.
3. **아이콘 시스템** — `lucide-react` 도입. 이모지(☰✕⚠)·수제 SVG 대체.
4. **서체** — 한글 가독 sans(Pretendard 권장) + 타이포 스케일.
5. **설비/상태 색 시스템(보라 제거)** — 설비=무채색(슬레이트 회색 4단계), **색=상태**(정상/점검/이상/정보). seed(assetTypes/cableCategories) + 프론트 색맵 + 캔버스 렌더러 일괄. 제어케이블 보라→청회색.
6. **전 화면 적용** — 로그인, 앱 셸(라이트 회색 사이드바+헤더), 워크스페이스 탭, 현황 테이블, 에디터 툴바, 모달.

### 비목표 (후속)
- 다크 모드 토글(토큰은 변수화하되 다크 테마는 후속).
- 기능 변경(동작은 유지, 외형만). 캔버스 좌표/로직 불변, 색/스타일만.
- 모바일 전용 레이아웃 최적화(반응형 기본만).

## 3. 디자인 토큰 (확정값)

### 색
```
/* 베이스/표면 */
--bg:        #f5f5f4   /* 앱 배경 (stone-100, 웜그레이) */
--surface:   #ffffff   /* 카드/패널 */
--surface-2: #fafaf9   /* 옅은 패널 (stone-50) */
--sidebar:   #f5f5f4   /* 라이트 회색 사이드바 (border #e7e5e4, 활성 #e7e5e4) */
--border:    #e7e5e4   /* 경계 (stone-200) */
--border-2:  #d6d3d1   /* 진한 경계 (stone-300) */
/* 텍스트 */
--text:      #1c1917   /* 본문 (stone-900) */
--text-2:    #57534e   /* 보조 (stone-600) */
--text-muted:#a8a29e   /* 흐림 (stone-400) */
/* 강조 (절제) */
--primary:   #15406b   /* 스틸블루 — 버튼/링크/선택 */
--primary-hover:#1d4ed8 /* 또는 #1a4e85 (구현 시 톤 확정) */
/* 상태 (색은 여기에만) */
--success:   #15803d   /* 정상 */
--warning:   #b45309   /* 점검/주의 */
--danger:    #b91c1c   /* 이상/알람 */
--info:      #1e40af   /* 정보 */
/* 상태 배경(배지) */
--success-bg:#dcfce7  --warning-bg:#fef3c7  --danger-bg:#fee2e2  --info-bg:#dbeafe
/* 설비 무채색 (ISA-101) */
--eq-1:#44403c  --eq-2:#78716c  --eq-3:#a8a29e  --eq-4:#d6d3d1
```
- Tailwind config 에 `theme.extend.colors` 로 매핑(`bg-surface`, `text-muted`, `border-default`, `text-primary` 등 시맨틱 유틸). 기존 `bg-blue-600`/`text-gray-500` 산재 → 시맨틱 토큰으로 점진 치환.

### 서체 / 타이포
- 폰트: **Pretendard**(한글) 폴백 시스템 sans. `@font-face` 또는 CDN.
- 스케일: `text-xs 12 / sm 13 / base 14 / md 15 / lg 17 / xl 20 / 2xl 24`, line-height 시멘틱. 기본 본문 14px(밀집 업무 화면).

### 간격 / 라운드 / 그림자
- 간격: 4/8/12/16/24/32 (Tailwind 기본 유지).
- 라운드: `--radius: 8px`(카드/버튼/인풋 통일), 배지 99px.
- 그림자: `--shadow-sm`(테두리+옅은), `--shadow-md`(드롭다운/모달). 과한 그림자 금지(평평·진중).

## 4. 컴포넌트 프리미티브 (`frontend/src/components/ui/`)
- `Button`(variant: primary/secondary/ghost/danger, size sm/md), `IconButton`, `Card`, `Badge`(status: success/warning/danger/info/neutral), `Input`·`Select`·`Textarea`, `Table`(헤더/행/zebra/hover/밀도), `Modal`/`Dialog`, `Tabs`, `Tooltip`. 모두 토큰 기반. lucide 아이콘 wrapping.
- 기존 인라인 Tailwind 화면을 점진적으로 이 프리미티브로 치환(big-bang 아님).

## 5. 설비/상태 색 시스템 (보라 제거 — 핵심)
- **설비**: assetType `displayColor` 를 **무채색**으로(랙=#44403c, OFD=#78716c, 분전반=#78716c, 모듈=#a8a29e 등 — 종류 구분은 회색 톤+아이콘/라벨). seed `assetTypes.ts` 보라(#a855f7/#8b5cf6/#7c3aed) 제거.
- **캔버스 렌더러**: 설비 fill=무채색, **테두리/상태점=상태색**(정상=success·점검=warning·이상=danger). 랙 모듈도 회색 fill + 카테고리는 옅은 톤/라벨. `equipmentRenderer`/`rack` 색 로직 수정.
- **현황/리스트**: 종류 색점(회색) + **상태 배지**(색)로 가독.
- **케이블**: 기능 표준색 유지(전력=빨강·통신=파랑·광=초록·접지=노랑). seed `cableCategories.ts` **제어케이블 보라(#a855f7)→청회색**(#64748b). `types/connection.ts CABLE_COLORS` 정합.
- **상태 배지(work order 등)**: IN_PROGRESS 보라(#purple)→info(파랑). `equipment/types`.

## 6. 화면별 타깃
- **로그인**: 좌측 다크 브랜드 패널(시스템 소개·로고) + 우측 라이트 폼(라벨·인풋·primary 버튼). 카드형 탈피.
- **앱 셸**: 라이트 회색 사이드바 + 상단 헤더(브랜드·브레드크럼·사용자). 시맨틱 토큰.
- **좌측 트리 패널 아이콘**(사용자 지적 — 핵심): 현재 이모지/수제 SVG 느낌의 노드 아이콘을 **lucide 비즈니스 아이콘**으로 — 본부=`Building2`, 사업소=`MapPin`/`Network`, 변전소=`Zap`/`Factory`, 층=`Layers`, 설비종류=`Server`/`HardDrive`/`Router` 등 일관 셋. 접기/펼치기 `ChevronRight`, 토글 ☰→`PanelLeft`. 크기·선·색 통일(토큰).
- **워크스페이스 탭**(현황/평면도/연결): primary 선택 강조, 일관 높이/간격.
- **현황 테이블**: 행 간격·타이포 위계↑, 종류 색점(회색)+상태 배지, lucide 액션 아이콘.
- **에디터 툴바**: lucide 아이콘 통일, 토글 상태 토큰, 정돈된 그룹.
- **모달/설계서/이력 패널**: Card/Modal 프리미티브, 일관 버튼.

## 7. 구현 접근 (점진)
1. **토대**: 토큰(CSS변수)+Tailwind 매핑+폰트+lucide 설치+프리미티브.
2. **설비/상태 색 시스템**(seed+색맵+렌더러) — 보라 제거(사용자 최우선).
3. **화면 롤아웃**: 로그인 → 앱 셸/사이드바 → 워크스페이스/테이블 → 에디터 툴바/모달. 화면마다 프리미티브+토큰 적용.
- 각 단계 빌드/타입체크 + 브라우저 확인(외형 변경이라 스모크 필수).

## 8. 테스트
- 단위: 프리미티브(variant/disabled 렌더), 색맵/상태 매핑 함수.
- **수동(브라우저)**: 로그인·셸·현황·평면도(설비 무채색+상태색, 보라 없음)·연결·설계서/이력 외형 확인. 기존 동작(저장·undo·토폴로지 등) 회귀 없음. 종류 구분 가독, 상태 한눈.
- 접근성: 텍스트 대비(WCAG AA), 상태색 + 아이콘 병기(색맹 대비).

## 9. 성공 기준
1. 디자인 토큰+프리미티브+lucide+서체 도입, 인라인 난립·이모지·수제SVG 대폭 축소.
2. **설비 무채색 + 색=상태**(보라 제거), 케이블 표준색.
3. 로그인~메인 전 화면이 일관된 업계표준 외형. 동작 회귀 없음.
