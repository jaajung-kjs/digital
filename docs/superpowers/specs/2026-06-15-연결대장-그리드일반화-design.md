# 연결 대장 그리드 일반화 (선번장·계통 통합) Design

**작성일:** 2026-06-15
**상태:** 설계 확정(사용자 승인) → 구현 플랜 대기

## 배경

연결 모델 대통일 이후, 선번장과 계통 두 그리드 뷰는 **완전히 동형의 구조**를 갖는다:

> **컨테이너 자산 → `connectionKind` 자식(=섹션) → 그 자식의 OUT 케이블(=행)**

| | 선번장 (`OfdFiberRegister`) | 계통 (`PanelCircuitView`) |
|---|---|---|
| 컨테이너 | OFD (placementKind 'OFD') | 분전반 (DIST) |
| 자식(섹션) | OFD-SLOT (connectionKind 'conduit') | 피더 (connectionKind 'distributor') |
| 행 | 슬롯의 OUT 케이블 = 코어 | 피더의 OUT 케이블 = CB |
| 행 출처 | 용량(OPGW cores)까지 빈 코어 포함, 1..N 연속 | 점유된 것만 |
| 컬럼 | 코어/근접자산/상대국측/용도/수용내역/융착/사용 | 번호/부하/용량/규격/SW |
| 편집 | 메타 인라인 편집(cable.specParams patch) | 읽기 전용 |
| 상대(far) | cableTrace 투영(대국 설비) | 1-hop(피더 반대편 부하) |
| 섹션 헤더 | 로컬-대국 변전소 | 피더명 |

두 컴포넌트가 컨테이너→자식→섹션→테이블 셸 + 행클릭 선택 로직을 **마크업·구조 80% 중복**으로 각자 구현한다. 새 슬롯형 대장(예: 접지계통)을 추가하면 또 한 벌 복붙해야 한다.

**문제:** "허브 + 번호 슬롯 + 점유/빈" 패턴이 코드로 일반화돼 있지 않아, 같은 구조를 매번 새로 만든다.

## 목표

컨테이너→자식→섹션→테이블 **공통 셸을 단일 제네릭 컴포넌트**로 추출하고, 자식별로 다른 부분(행 출처·컬럼·편집·far·헤더)은 **descriptor(설정 객체)** 로 공급한다. 선번장·계통을 각자의 descriptor로 재구성해 **동작을 100% 보존**하면서 중복을 제거한다. 그러면 향후 슬롯형 대장(접지 등)은 **descriptor 하나 추가**로 끝난다.

**비목표:** 모델/백엔드 변경 없음(순수 프론트 리팩토링). 접지계통 자체 구현(백엔드 자산종류·케이블카테고리 필요 → 별도 벌티컬). 이번엔 descriptor 패턴만 준비.

## 아키텍처

### 제네릭 컴포넌트 `ConnectionRegisterGrid`
```
<ConnectionRegisterGrid substationId descriptor={fiber | power} />
  └ descriptor.selectContainers(assets, substationId) 별:
       (descriptor.containerHeader 있으면 컨테이너 헤더 렌더)
       └ parentAssetId===container.id && connectionKind===descriptor.childKind 자식별 섹션:
            └ descriptor.buildSection(child, ctx) → { title, usedLabel, rows }
                 헤더(title + usedLabel) + 테이블(descriptor.columns) + rows(행클릭→descriptor.onRowClick)
```
- 컨테이너 0개면 `descriptor.emptyMessage`.
- 행클릭 → `useSelectionStore.setSelectedAssetId(descriptor.onRowClick(row, child))`.
- ctx = `{ assets, cables, graph }` (graph = `useTraceGraph().graph`; 컴포넌트가 항상 구독, descriptor가 안 쓰면 무시 — React Query dedup이라 비용 미미).
- 컴포넌트는 **순수 구조/셀렉션/렌더 셸만** — 도메인 로직 0.

### `RegisterDescriptor` (렌더-함수 컬럼)
```ts
interface RegisterCtx { assets: Asset[]; cables: Cable[]; graph: TraceGraph | null }
interface RegisterColumn<Row> { label: string; width?: string; cell(row: Row): ReactNode }
interface RegisterSection<Row> { key: string; title: string; usedLabel: string; rows: Row[] }

interface RegisterDescriptor<Row> {
  emptyMessage: string;
  selectContainers(assets: Asset[], substationId: string): Asset[];
  childKind: 'conduit' | 'distributor';
  containerHeader?(container: Asset, ctx: RegisterCtx): string | null;
  buildSection(child: Asset, ctx: RegisterCtx): RegisterSection<Row>;
  columns: RegisterColumn<Row>[];
  onRowClick(row: Row, child: Asset): string | null;  // 선택할 assetId
}
```
- **컬럼 = 렌더 함수** — 셀 안에서 정적 텍스트든 `<input>`/`<select>`(편집)든 자유. 편집/비편집/far 투영을 type 분류 없이 descriptor가 자연 흡수(사용자 선택).
- 자식 필터는 컴포넌트가 일반 규칙(`parentAssetId===container && assetType.connectionKind===childKind`)으로 처리 — descriptor는 `childKind`만 선언.

### 두 descriptor (기존 로직 이전 — 동작 보존)

**`fiberRegisterDescriptor`** (구 `OfdFiberRegister` 로직):
- `selectContainers` = `assets.filter(a => a.assetType?.placementKind==='OFD' && a.substationId===subId)`.
- `childKind='conduit'`. `containerHeader` 없음(OFD명은 'OFD'라 무의미).
- `buildSection(slot, ctx)` = `buildSlotCoreRows(slot, ctx.cables, ctx.graph)` 로 행 + 헤더 title=`로컬sub - 대국sub`(graph.subNameById + remoteSlotSubstation), usedLabel=`사용 {used}/{rows.length}`.
- `columns` 7개: 코어(static), 근접자산(graph.nameById lookup), 상대국측(row.farName), 용도/수용내역(`<input>` onBlur→commitMeta), 융착/사용(`<select>` onChange→commitMeta). commitMeta = `wc.patch('cables', cableId, {specParams:{...prev,[field]:value}})`(빈 코어=cableId null→disabled).
- `onRowClick(row, ofd)` = `row.nearAssetId ?? ofd.id`.
- Row 타입 = `SlotCoreRow`(기존).

**`powerRegisterDescriptor`** (구 `PanelCircuitView` 로직):
- `selectContainers` = `assets.filter(a => a.substationId===subId && (a.assetType?.code==='DIST' || a.assetType?.placementKind==='DIST'))`.
- `childKind='distributor'`. `containerHeader(panel)` = `panel.name`.
- `buildSection(feeder, ctx)` = ctx.cables 중 feeder OUT 케이블 → CB행(점유만): {cableId, loadAssetId, loadName(nameById), cbNumber/capacity/switchState(specParams), spec(categoryName)}. usedLabel=`사용 {ON수}/{rows.length}`.
- `columns` 5개: 번호/부하/용량/규격/SW(전부 static 텍스트, ON이면 점 표시).
- `onRowClick(row, feeder)` = `row.loadAssetId ?? feeder.id`.
- Row 타입 = `CbRow`(기존).

### 재배선
- `FiberRegisterView`(사이드패널 래퍼 유지) 내부 = `<ConnectionRegisterGrid substationId descriptor={fiberRegisterDescriptor} />`.
- `PowerView`(사이드패널 래퍼) 내부 = `<ConnectionRegisterGrid substationId descriptor={powerRegisterDescriptor} />`.
- 구 `OfdFiberRegister.tsx`·`PanelCircuitView.tsx` 본문 삭제(로직은 descriptor로 이전; commitMeta/Cb행 빌더 등은 descriptor 파일 또는 인접 헬퍼로).

## 데이터 흐름
```
useEffectiveAssets / useEffectiveCables / useTraceGraph
  → ConnectionRegisterGrid(descriptor)
     → descriptor.selectContainers → 자식(childKind) → buildSection → rows
     → columns.cell(row) 렌더(정적/편집)
     → 행클릭 → selectionStore (사이드패널 연결탭 = 기존 흐름)
편집(선번장) → cell 내 commitMeta → wc.patch('cables', …) → 전역 overlay → 커밋
```

## 엣지/에러 처리
- **graph 로딩 중**: far/대국명은 `?? fallback`(기존 동작 동일). buildSlotCoreRows/remoteSlotSubstation은 graph null 안전.
- **컨테이너/자식 0개**: emptyMessage / 빈 섹션(렌더 안 함).
- **빈 코어 행 편집**: cableId null → 입력 disabled(기존과 동일).
- **distributor 용량 무한**: 점유만 표시(buildSection이 빈 행 안 만듦). conduit은 용량까지 빈 행(buildSlotCoreRows). → 행 출처가 descriptor에 캡슐화돼 connectionKind 차이를 흡수.

## 테스트 전략
- **순수 단위**: 각 descriptor의 `buildSection`(rows·usedLabel·title) + `selectContainers`/`onRowClick`. (컬럼 cell은 렌더라 컴포넌트 테스트에서.)
- **컴포넌트**: 기존 `FiberRegisterView.test.tsx`(선번장 렌더·행클릭·편집 patch)가 새 셸 위에서 그대로 통과 = 동작 보존 증명. 계통은 신규 경량 테스트(섹션·행 렌더) 1개.
- **빌드/타입**: 프론트 tsc + build + 전체 vitest green.
- **수동 스모크**: 선번장·계통 둘 다 기존과 시각·편집·선택 동일.

## 비목표 (YAGNI)
- 접지계통 구현(별도 벌티컬). 컬럼 정렬/필터/페이지네이션(현재 없음). 행 가상화. descriptor의 type-드리븐 컬럼(렌더-함수로 결정).

## 단계 분해(플랜 예고)
1. 제네릭 `ConnectionRegisterGrid` + `RegisterDescriptor` 타입 + 셸(컨테이너→자식→섹션→테이블→선택).
2. `fiberRegisterDescriptor`(구 OfdFiberRegister 이전) + FiberRegisterView 재배선 + OfdFiberRegister 삭제.
3. `powerRegisterDescriptor`(구 PanelCircuitView 이전) + PowerView 재배선 + PanelCircuitView 삭제.
4. 검증 + 최종 리뷰.
