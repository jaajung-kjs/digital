# 연결도 단일 함수(Connection Diagram) Design

**작성일:** 2026-06-16
**상태:** 설계(모델 검증 완료: 단일 규칙으로 전력/광/네트워크/다단/타층 일관) → spec 리뷰 → 구현 플랜
**큰 그림:** 연결탭의 경로 트리를 **휴리스틱 조건분기 없이 단 하나의 함수**로 모든 케이스에서 정확히 그린다. 현재 `projectTrace` 가 **시드 케이블의 passive 끝**을 루트로 잡아 관점마다 방향이 달라지는 문제를, **"보는 자산이 속한 변전소 안에 공급 원점이 있으면 그 원점을, 없으면 보는 자산을 루트로"** 라는 단일 규칙으로 대체한다.

---

## 1. 배경 / 현재 문제

- **현재:** `traceProjection.ts:projectTrace(seedCableId)` 가 시드 케이블의 passive 끝을 `start` 로 잡고 BFS 트리(`tree`)를 만든다. 루트가 **클릭한 케이블**에 따라 달라져, 같은 연결을 다른 자산에서 보면 방향/모양이 달라진다("어색"의 원인). 또 추적이 변전소 경계를 무시하고 대국까지 선형으로 펼쳐진다.
- **재사용 자산(유지):** `cableTrace`(역할·채널 인지 순회), `traceGraph`(전역+staged 오버레이, `nameById`/`subNameById`/`parentById`/`codeById`), `NetworkTopologyModal`(전체 토폴로지=[상세]), `usePathHighlightStore`(하이라이트셋), `PathTraceDetail`(트리 렌더 — 이번에 루트만 바뀜), `buildSelfSideChecker`/`buildEndpointNameResolver`.
- **핵심 분리:** **연결탭 트리 = 변전소 스코프 + 규칙 루트(신규)**, **[상세] 토폴로지 모달 = 전체 망(기존 그대로)**. 대국/링 전체는 [상세]에서만.

## 2. 목표 / 비목표

**목표**
- 연결탭이 보는 자산(+보유자산)의 연결을 **종류별로**, **단일 규칙으로 루트를 정한 트리**로 보여준다.
- 설비 관점 = 나에게 오는 공급 경로(출발→경유→나), 분전반/상위 관점 = 나에게 연결된 설비들 한눈에. 둘 다 같은 트리(강조 노드만 다름).
- 경로를 **분기까지 텍스트로 정확히**(PathTraceDetail 트리 렌더 재사용).
- **다양한 케이스 시드 데이터**로 모든 패턴 검증.

**비목표(YAGNI)**
- [상세] 토폴로지 모달 변경(전체 망은 그대로).
- 종류별 전용 분기 함수/조건(단일 함수가 규칙으로 처리).
- 케이블 생성/편집(연결탭 읽기 전용).
- 백엔드 변경(시드 데이터 추가 제외).

## 3. 단일 규칙 (핵심)

> **① 스코프:** 보는 자산의 **변전소** 안에서만 추적(층 무관). 변전소를 벗어나는 엣지(대국 OPGW 등)는 **끝점 라벨**(경로명·#코어)로 끊는다.
> **② 방향:** 케이블 역할(IN=공급받음/상류, OUT=공급함/하류)로 방향. 역할이 없으면 무방향.
> **③ 루트:** 변전소 안에 **공급 원점**(= IN 이 없는 순수 공급원이면서 통과설비(distributor/conduit)가 아닌 진짜 source)이 있으면 **그 원점이 루트**. 없으면(공급이 대국=밖에서 오거나 무방향) **보는 자산(self-set 연결점)이 루트**.
> **④ 분배:** 루트에서 아래로, 통과설비 fan-out 은 분기로.
> **⑤ 하이라이트:** 보는 자산(self-set)을 강조. 캔버스는 현재 층 노드만 점등(기존).

검증(모든 케이스가 ①~⑤에서 자연히 도출 — 조건분기 없음):

| 케이스 | 변전소 안 공급 원점? | 결과(루트 → …) |
|---|---|---|
| 전력 다단(충전기→UPS→분전반→부하들) | 있음(충전기) | `충전기 → UPS → 분전반 ├ 피더A→{단말1,단말2} └ 피더B→{단말3}` (누가 열든 동일, 강조만 다름) |
| 전력 공급원 타 층 | 있음(같은 변전소) | 동일 source-root(텍스트엔 다 나오고 캔버스는 현재 층만 점등) |
| 광 단말 | 없음(대국=타 변전소) | `단말 → 춘천-남춘천 #24`(슬롯=경로 라벨, 끝) |
| 광 OFD | 없음 | `슬롯S ─분기─ {단말광1, 단말광2}`(슬롯별) |
| 네트워크 무방향 체인(설비1-2-3-4, 설비3 관점) | 없음(무방향) | `설비3 ├ 설비2→설비1 └ 설비4` |

## 4. 알고리즘 — `buildConnectionDiagram(...)`

신규 순수 함수. 입력: `{ graph, assets, assetId, cableType(or group) }`. 출력: 종류별 `DiagramComponent[]`(컴포넌트=루트 트리 1개).

```ts
export interface DiagramNode {
  id: string;            // collapse 후 노드(슬롯→OFD 접힘)
  label: string;         // 표시명(대국 경계면 경로명 #코어)
  kind: 'asset' | 'boundary'; // boundary = 변전소 밖 끝점(대국)
  isSelf: boolean;       // self-set 강조
  isOrigin: boolean;     // 루트(공급 원점) 여부
  edgeFiber: boolean;    // 부모→이 노드 엣지가 OPGW(점선)
  children: DiagramNode[];
}
export interface DiagramComponent {
  groupKey: string; groupLabel: string; groupColor: string | null;
  root: DiagramNode;     // 규칙으로 정한 루트
  cableIds: string[];    // 하이라이트용(컴포넌트 전체)
  nodeIds: string[];     // 하이라이트용(원시 노드)
}
```

단계:
1. **self-set / 변전소:** `isSelf = buildSelfSideChecker(assets, assetId)`. `subId = substationId(assetId)`. 그래프 노드별 `substationId` 필요(§9 그래프 보강).
2. **종류별 후보 케이블:** self-set 에 닿고, 해당 종류(group)인 케이블.
3. **변전소 스코프 컴포넌트 추적:** self-set 시드에서 `cableTrace`(역할·채널 인지)로 컴포넌트를 모으되, **다른 변전소 노드로 가는 엣지는 traverse 하지 않고 boundary 로 기록**(끝점 라벨 = 경로명/대국 #코어). 같은 변전소의 다른 층 노드는 정상 traverse.
4. **방향 그래프:** 컴포넌트 케이블로 유향 인접(역할 OUT=공급→하류, IN=상류). 역할 없으면 무방향(양방향).
5. **공급 원점 탐지:** 컴포넌트에서 `kindOf(node) ∉ {distributor, conduit}` 이고 **상류(IN) 엣지가 변전소 안에 없는** 노드 = 원점 후보. 있으면 결정적(이름/ID 순) 1개를 **루트**. 없으면 **루트 = self-set 연결점**(self 노드 중 케이블이 닿는 것; 여러 개면 각각 한 트리 = OFD 슬롯별).
6. **트리 빌드:** 루트에서 흐름 방향(원점→하류, self-root 는 바깥 방향)으로 BFS 트리. conduit 슬롯→부모 OFD 접기 + 접힘 중복은 손주 승격(기존 `buildTree` 로직 재사용). boundary 노드는 leaf(`kind:'boundary'`, label=경로명#코어). `isSelf`/`isOrigin`/`edgeFiber` 마킹.
7. **종류별 그룹화:** `makeCategoryGroupOf`(기존, cableType→displayGroup 폴백 포함)로 그룹 키/라벨/색.

**단일성 보장:** 모든 분기(전력/광/네트워크)는 §3 규칙의 데이터(역할 유무·원점 유무·변전소 경계)에서 갈리고, 함수 본문엔 종류별 if 가 없다. cableTrace 가 이미 역할/채널로 방향·격리를 처리하므로 프론트는 케이스를 나누지 않는다.

## 5. 렌더링 / UI

- **연결탭(`AssetConnectionsTab`):** 종류별 그룹 헤더 + **컴포넌트별 한 트리**(요약). 트리는 `DiagramNode` 를 §렌더러로 그린다(현 `PathTraceDetail`의 트리 렌더 일반화 — 선형은 한 줄, 분기는 분기 노드 아래 들여쓰기, `isSelf` 강조, boundary 는 점선/뱃지). 행 클릭 → 컴포넌트 `cableIds`로 하이라이트.
- **[상세]:** 기존 `NetworkTopologyModal`(전체 망) 그대로 — 대국·링은 여기서.
- **하이라이트:** self-set 강조 + 컴포넌트 노드(현재 층만 캔버스 점등). `usePathHighlightStore` 에 `startDiagram(component)` 추가(또는 기존 `startTrace` 를 컴포넌트 cableIds 로 호출).
- **대국 표시:** boundary leaf = `경로명 #코어`(예: `춘천S/S-남춘천S/S #24`), 회색 점선 뱃지. 클릭 시 [상세]에서 전체.

## 6. 시드 데이터 (테스트 변전소)

`backend/prisma/seed-conn-cases.sql`(또는 seed 스크립트) — 독립 변전소 `sub-연결테스트` + 대국 1개. 패턴:

- **P1 전력 다단 분배(같은 층):** `충전기 → UPS(distributor) → 분전반(distributor) → [피더A(distributor)→{단말1,단말2}], [피더B→{단말3}]`. 역할 IN/OUT 채움.
- **P2 공급원 타 층:** `충전기(B1층) → 분전반(1층) → 단말4(1층)`.
- **P3 광 단말+OFD+대국:** 로컬 `OFD` + 슬롯 `S`(conduit), `S ─OPGW(IN-IN)─ 원격OFD(sub-대국)`, `S ─OUT#1─ 단말광1`, `S ─OUT#2─ 단말광2`.
- **P4 네트워크 무방향 체인:** `설비N1—N2—N3—N4`(LAN, 역할 null).
- **P5 엣지:** 같은 두 끝에 전원+접지 케이블 동시 / 입력만 있는 피더 / categoryId 없는 미분류 케이블.

검증 카운트 SELECT 포함. 개발 DB(`docker compose -f docker-compose.dev.yml`)에 적용해 수동 확인.

## 7. 통합 / 파일 영향

- **신규:** `frontend/src/features/connections/connectionDiagram.ts`(`buildConnectionDiagram` + 타입, 순수, 단위테스트). 렌더러는 `PathTraceDetail` 트리 부분을 `DiagramNode` 기반으로 일반화(또는 신규 `DiagramTree` 컴포넌트).
- **수정:** `useAssetConnections.ts` → 컴포넌트 트리 산출로 전환(기존 `buildConnectionGroups` 의 그룹/색 로직 재사용, 행 산출만 트리로). `AssetConnectionsTab.tsx` 트리 렌더. `pathHighlightStore` 에 컴포넌트 하이라이트 진입점.
- **그래프 보강:** `traceGraph` 노드에 `substationId`(스코프 판정용) 노출. `cableTrace`/`projectTrace`(토폴로지)는 **불변**.
- **시드:** `backend/prisma/seed-conn-cases.sql`(+ npm 스크립트).
- **재사용(불변):** `NetworkTopologyModal`, `cableTrace`, `makeCategoryGroupOf`, `buildSelfSideChecker`/`buildEndpointNameResolver`.

## 8. 테스트 전략

- **`buildConnectionDiagram`(단위):** P1~P5 각각 — 루트 결정(원점 vs self), 분기 구조, 변전소 경계 boundary leaf(대국 라벨), 다층 source-root, 무방향 양방향 분기, 종류 분리. **같은 컴포넌트를 여러 자산에서 호출 시 동일 트리 + isSelf 만 다름**을 명시 검증.
- **렌더러(컴포넌트):** 트리/분기/boundary/강조 렌더, 행 클릭 → 하이라이트 진입점 호출.
- **빌드/타입:** `npx tsc` + 전체 `npx vitest` green(frontend 디렉터리).
- **수동 스모크:** 시드 적용 후 P1~P4 각 관점에서 연결탭 — 규칙대로 루트/분기/대국 라벨, 하이라이트, [상세] 토폴로지.

## 9. 엣지 / 에러

- **공급 원점 다수:** 결정적(이름/ID 순) 1개 선택(드묾 — 다중 인입). 필요 시 가상 루트로 묶기는 차기.
- **원점 없음 + self 다중 연결점(OFD 슬롯들):** self-set 연결점별 트리(슬롯별 행).
- **링/사이클(전력 루프·광 링 일부가 변전소 안):** BFS 트리 back-edge 제거(기존), 변전소 밖 링은 boundary 로 끊김.
- **무방향 + 사이클(네트워크 메시):** BFS 트리(임의 back-edge 제거) — 한눈 요약 우선, 전체는 [상세].
- **staged(미저장) 케이블:** `graph.cables` 오버레이 포함 → 그대로 반영.
- **연결 0개:** 빈 상태.

## 10. 핵심 불변식

- 함수 본문에 **종류별 조건분기 없음** — 방향/원점/경계는 데이터(역할·connectionKind·substationId)에서.
- 같은 변전소 컴포넌트는 **누가 열든 같은 트리**(강조 노드만 다름).
- [상세] 토폴로지(전체 망)는 **불변**.
