# 연결 모델 대통일 (Universal Connection Model) 설계

**작성일:** 2026-06-13 (긴 설계 대화의 수렴본)
**성격:** 대규모·다서브시스템 마이그레이션 — 단계(슬라이스)로 분해해 각 단계가 독립 출하·검증되게 한다.

---

## 1. 한 줄 요약

**OFD·분전반은 그냥 컨테이너 자산. 그 안의 슬롯(conduit)·피더(distributor)는 connectionKind 가진 자식 자산. 모든 연결은 IN/OUT role + number 가진 케이블. 추적은 범용 cableTrace 한 함수. 세 그리드(현황·선번장·계통)는 같은 패턴의 투영, 평면도/토폴로지도 그 추적 결과의 투영.** → FiberPath·FiberCore·usePortStatus·buildPortStatuses·OfdPortPicker·포트격리BFS·분리된 trace들 전부 소멸.

## 2. 모델 (확정)

### 2.1 자산 = 컨테이너 + connectionKind 자식 (동일 패턴)
```
분전반(일반자산, 컨테이너) → 피더(자식, connectionKind=distributor)
OFD   (일반자산, 컨테이너) → 슬롯(자식, connectionKind=conduit)   ← 슬롯 1개 = OFD↔대국 경로 1개
```
- 컨테이너(OFD/분전반/랙)는 평범한 자산. 자식이 여러 개 = 그냥 `parentAssetId`(합성).
- `AssetType.connectionKind`: `none | distributor | conduit` (이미 컬럼·시드 존재; OFD=현재 conduit 표식).

### 2.2 모든 연결 = 케이블 (IN/OUT role + number)
이미 있는 `Cable.sourceRole/targetRole`(IN/OUT) + 신규 `Cable.number`(범용 슬롯/포트/CB 번호, = fiberPortNumber 일반화).
```
distributor(피더):  input  ─케이블─ 충전기/상용      (IN, 1개)
                    output ─케이블─ 부하들           (OUT, N개)
conduit(슬롯):      input  ─OPGW 케이블─ 대국 슬롯의 input   (IN끼리, 1개)
                    output ─케이블(number=K)─ 로컬 설비      (OUT, N개)
```
- **OPGW = 슬롯 input끼리 잇는 케이블 1개** (24코어라도 24개 아님). 카테고리=OPGW/OJC 등 실제 자재.
- **코어 N개 = 슬롯의 output 케이블 N개**(각 `Cable.number=K`). 메타(용도/수용내역/융착) = 그 output 케이블 specParams(전원 specTemplate 패턴과 동일). 빈 코어 = 슬롯 capacity에서 도출.
- 변전소-변전소 연결도 그냥 케이블(전역 워킹카피).

### 2.3 범용 cableTrace = 함수 1개
같은 cableType 연결을 순회해 **전체 물리 경로** 반환. 노드 connectionKind로 *다음 홉 규칙만* 분기:
- **none/passive**: 닿은 케이블로 통과(분기 포함).
- **distributor**: output에서 왔으면 → input(IN)으로 수렴(상류). (전원 상류추적)
- **conduit**: output#K에서 왔으면 → input(OPGW) 건너 대국 슬롯 → 대국 output#K (같은 number). (광경로)
사이클=링. 한 함수가 광경로·전원상류·평면경로·토폴로지를 전부 산출. (스코프 파라미터: start, cableType.)

### 2.4 뷰 = 투영 (경계는 placement, 직교)
- **현황** = 자산 목록.
- **선번장** = OFD의 자식 슬롯별 output 케이블(번호) + 메타 나열. **계통** = 분전반의 자식 피더별 output 케이블 나열. → **동일 컴포넌트**(컨테이너 자산 + 자식(connectionKind) + 그 output 케이블 그리드).
- **평면도 하이라이트** = trace 결과 중 *현재 도면에 배치된(placeable) 부분만* 그림(다른 층/미배치/타 변전소면 거기서 멈춤 — placement 판정, 일반).
- **토폴로지** = trace 결과 전체 그림.
- 경계 = "노드가 현재 도면에 있나"(placement) — connectionKind와 **무관**.

## 3. 소멸하는 것 (순 감소)
`FiberPath`(model/service/controller/routes/REST 5개) → 슬롯 자산 + OPGW 케이블 / `FiberCore` → output 케이블 specParams / `fiberPortNumber` → `Cable.number` / `usePortStatus`·`buildPortStatuses`·`OfdPortPicker`·포트격리 BFS·`tracePathToRoot`/별도 fiber BFS → 범용 cableTrace + 케이블 연결. (`composeFiberPaths`, `useFiberPaths`, fiber `merge`, 전역 `/fiber-paths` 등 부속도.)

## 4. 단계 분해 (각 단계 독립 출하·검증)

> 위험: 지금 *잘 도는* 트레이서/토폴로지/usePortStatus를 대체한다. 그래서 **범용 trace가 기존 동작을 재현함을 테스트로 증명한 뒤** 레거시를 지운다. "추가→증명→전환→삭제" 순.

- **P1 — 모델 토대**: `Cable.number` 컬럼(+커밋 스키마/서비스, fiberPortNumber와 병행) + 슬롯 AssetType(conduit) + OPGW/OJC CableCategory. (마이그레이션 아님, 추가만.)
- **P2 — 범용 cableTrace**: `connectivity trace`(none/distributor/conduit) 순수 함수 + 광/전원/평면 경로를 **기존과 동일하게 재현**하는 테스트. 기존 trace는 아직 유지(병행).
- **P3 — 뷰 투영 전환**: 평면도 하이라이트·토폴로지·선번장·계통을 범용 trace 결과의 투영으로 재배선. (선번장/계통 단일 그리드 컴포넌트.)
- **P4 — 데이터 마이그레이션**: FiberPath→슬롯 자산 + OPGW output/ input 케이블; fiberPortNumber→Cable.number; FiberCore→output 케이블 specParams. (14 OFD + 기존 데이터; 검증 쿼리.)
- **P5 — 레거시 삭제**: FiberPath/FiberCore/usePortStatus/OfdPortPicker/포트격리BFS/분리 trace 제거. grep=0 확인.
- **P6 — 편의 레이어**: 슬롯 만들 때 N output 자동생성 + 대국 슬롯 자동설치(별도 프론트 헬퍼, 모델과 분리).

각 단계 = 자체 spec 세부 + plan + subagent 실행 + 검증. P1~P3은 추가/병행이라 저위험, P4~P5가 핵심 전환(고위험, 신중).

## 5. 확정해둘 구현 결정 (스펙 세부에서 못 박음 — 기본값/추천)
1. **슬롯 capacity(24/48)** = 슬롯 자산 속성. 빈 코어 = capacity − output 수 도출.
2. **슬롯 희소 materialize**(쓰는/메타 있는 코어 output만 케이블; 슬롯 자산은 경로당 1개) — 랙/FiberCore 일관.
3. **conduit 트레이스 number 매칭** = output 케이블 `Cable.number` + 슬롯 input(OPGW) 페어로.
4. **OPGW/OJC = CableCategory**(실제 자재). 변전소간 = 일반 케이블.
5. **선번장 메타** = output 케이블 specParams (전원 CB 메타와 동형). 빈 코어 주석은 후속(필요 시 슬롯 속성).
6. **경계 = placement**(floorAnchor) — connectionKind 직교.

## 6. 리스크 / 비-목표
- 리스크: cross-substation 도출·링 토폴로지·포트격리를 범용 trace로 재현하는 정확성(P2 테스트로 게이트). 데이터 마이그레이션(P4) 무손실.
- 비-목표(후속): 전체 SLD 자동작도, 접지, 변전소간 경로 정의 UI 확장.
