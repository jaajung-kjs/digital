# 연결 대장 그리드 셀 편집 일관화 (호버 연필 인라인 편집) Design

**작성일:** 2026-06-15
**상태:** 설계 확정(사용자 승인) → 구현 플랜 대기

## 배경

연결 대장 그리드(선번장·계통)를 단일 `ConnectionRegisterGrid` + descriptor 로 구조 통일했으나(머지 완료), **편집 UX가 일관되지 않다**:

1. **계통뷰엔 편집 진입점이 아예 없음.** 케이블 정보는 사이드패널에서 못 고친다(인스펙터는 자산만 — 케이블 안 보임). 케이블 편집은 그리드 전용인데, 계통 그리드는 읽기 전용이라 **CB 정보(번호/용량/SW)를 고칠 방법이 0.**
2. **선번장은 텍스트박스를 처음부터 노출.** 항상 보이는 `<input>`/`<select>` — 사이드패널/현황의 "호버→연필→편집" 패턴과 불일치.
3. **Enter가 입력을 커밋 안 함.** 그리드 입력에서 Enter 무반응, 다른 곳 클릭(blur)해야 반영. (로직은 있는데 UX가 따로 놂.)

기준 패턴은 이미 존재한다 — `AssetInspector` 의 `Field`(`AssetInspector.tsx:64-95`): 읽기=텍스트, **호버→연필(✎)**, ✎ 클릭→인라인 입력, **Enter→blur→커밋**. 단 (a) 인스펙터 안에 박혀 재사용 불가, (b) Esc 취소 없음.

## 목표

`AssetInspector.Field` 의 **호버 연필 + Enter 커밋** 인라인 편집을 **공유 `EditableField` 컴포넌트로 추출**하고, 두 그리드 descriptor 가 이를 써서 케이블 필드를 **그리드에서 직접·일관되게** 편집한다. 계통도 편집 가능해지고(빠진 진입점), 선번장의 항상-노출 입력은 호버-연필로 통일된다. **행클릭→사이드패널 의도는 보존**(✎ 만 편집 진입).

**비목표:** 케이블을 사이드패널에서 편집(현 구조 유지 — 그리드 전용). date/textarea/status 등 인스펙터 고유 변형의 일반화(text/select만). 새 컬럼/정렬/필터.

## 핵심 통찰 — 행클릭 긴장 자동 해소

`Field` 패턴의 읽기모드는 **평범한 텍스트 + 작은 호버 연필**이다. 그리드 셀에 적용하면:
- 셀/행 클릭(✎ 제외) → **사이드패널**(의도 보존).
- **✎ 만** 인라인 편집 진입(`stopPropagation`).

→ "모든 필드가 클릭=편집이라 패널 열 곳이 없어진다"는 우려가 사라진다(✎ 외엔 전부 패널).

## 아키텍처

### 공유 컴포넌트 `EditableField`
**위치:** `frontend/src/features/assets/components/EditableField.tsx`(인스펙터 옆 — 인스펙터도 쓰므로).

```tsx
interface EditableFieldProps {
  value: string;                               // 현재 값(읽기 표시·편집 초기값)
  type?: 'text' | 'select' | 'date';
  options?: { value: string; label: string }[]; // select 일 때
  placeholder?: string;
  disabled?: boolean;                          // 편집 불가(✎ 안 보임) — 빈 코어 등
  display?: (value: string) => ReactNode;      // 읽기 표시 커스텀(예: SW 점·자동(사용))
  ariaLabel?: string;
  onCommit(value: string): void;               // 값 변경 시(변경된 경우만)
}
```
- **읽기모드:** `group` 셀 안에 텍스트(`display(value)` 또는 `value || '—'`) + 호버 시 그 셀에만 ✎(`opacity-0 group-hover:opacity-100 …`, 인스펙터 PENCIL 클래스 재사용). disabled 면 ✎ 미표시.
- **편집모드:** ✎ 클릭(`stopPropagation`)→ `editing=true`. `type='text'` → `<input autoFocus defaultValue>`; `type='select'` → `<select autoFocus value + options>`. 인스펙터 `INLINE_INPUT` 스타일 재사용.
- **커밋/취소:**
  - text: `onBlur` → `commit(value)`(변경 시 `onCommit`) + `setEditing(false)`. `onKeyDown` Enter → `blur()`. **Esc → 복원(원래 value 유지)·`setEditing(false)`**(인스펙터에 없던 개선).
  - select: `onChange` → `onCommit(value)` + `setEditing(false)`. Esc → 취소.
  - date: `onChange` → `onCommit(value)`(인스펙터 date 와 동일 — 선택 즉시 반영) + `setEditing(false)`. Esc → 취소.
- 모든 인터랙션 핸들러에 `stopPropagation`(행클릭 차단).

### 두 descriptor 가 EditableField 사용
**`fiberRegisterDescriptor`** — 편집셀을 `<EditableField>` 로 교체(현재 항상-노출 input/select):
- 용도/수용내역 = `type='text'`, `onCommit=(v)=>commitMeta(cableId,'purpose'|'circuitText', v||null)`.
- 융착 = `type='select'`(— / 융착 / 패치).
- 사용 = `type='select'`(자동 / 사용 / 미사용), `display` 로 `자동(사용)` 표기.
- 빈 코어(cableId 없음) = `disabled`.
- 코어/근접자산/상대국측 = 읽기(현 그대로).

**`powerRegisterDescriptor`** — 읽기 전용 → 편집 추가:
- 번호(cbNumber)/용량(capacity) = `type='text'`, `onCommit=(v)=>commitMeta(cableId,'cbNumber'|'capacity', v||null)`.
- SW(switchState) = `type='select'`(ON / OFF), `display` 로 ON 점 표시(현 CbRowView 룩 보존).
- 부하/규격 = 읽기(부하=자산, 규격=카테고리 — 비 specParams).
- CbRow 에 `cableId` 노출(이미 있음) → commit 대상. `commitMeta` 를 power descriptor 에도 추가(`patch('cables', cableId, {specParams:{...prev,[field]:value}})` — fiber 와 동일).

### 인스펙터 `Field` 도 `EditableField` 채택
`AssetInspector.tsx` 의 `Field`(text/date)를 **`EditableField`(type='text'|'date')로 완전 교체** → 단일 컴포넌트·Esc 일관. `DescField`(textarea)/`StatusField`(pill) 는 그리드에서 안 쓰므로 인스펙터 고유로 유지. 인스펙터 `AssetInspector.test` 로 동작 보존 검증(이름/담당자 text, 설치일 date 편집).

## 데이터 흐름
```
EditableField (읽기→호버✎→편집→Enter/blur 커밋, Esc 취소)
  → onCommit(value) → descriptor commitMeta(cableId, field, value)
     → wc.patch('cables', cableId, {specParams:{...prev,[field]:value}})  (specParams 기존 키 보존)
     → 전역 overlay → 저장(커밋 바)
행/텍스트 클릭(✎ 제외) → onRowClick → selectionStore → 사이드패널(연결탭)  [기존]
```

## 엣지/에러 처리
- **행클릭 vs ✎:** ✎ + 입력 컨트롤 전부 `stopPropagation` → 편집이 행클릭(패널) 안 트리거.
- **빈 코어/비편집:** disabled → ✎ 미표시 → 읽기만(행클릭=패널).
- **Enter:** text 는 blur 로 커밋. select 는 변경 즉시 커밋.
- **Esc:** 편집 취소·원값 복원(커밋 안 함).
- **specParams 병합:** commitMeta 가 `{...prev, [field]: value}` → `__migration` 등 기존 키 보존(P4 마커 안전).
- **연속 편집:** 한 셀 편집 중 다른 셀 ✎ 클릭 → 첫 셀 blur 커밋 후 둘째 진입(자연스러운 포커스 이동).

## 테스트 전략
- **EditableField 단위(컴포넌트):** 읽기 표시, 호버 ✎ 노출, ✎ 클릭→입력, Enter→onCommit(변경 시만), Esc→취소(onCommit 미호출), select onChange→onCommit, disabled→✎ 없음, stopPropagation(행클릭 미발생).
- **descriptor 동작 보존/추가:** 기존 `FiberRegisterView.test`(편집 patch) 가 `EditableField` 경유로 통과(편집 진입이 ✎ 클릭으로 바뀐 만큼 테스트의 편집 트리거 수정 — 단언(patch 호출)은 보존). 계통 편집 신규 테스트(✎→입력→commitMeta patch).
- **인스펙터:** `AssetInspector.test` 통과(Field→EditableField 교체 후 동작 보존).
- **빌드/타입:** 프론트 tsc + build + 전체 vitest green.
- **수동 스모크:** 선번장·계통 둘 다 호버 ✎ → 편집 → Enter 커밋 → 저장; 행클릭 → 패널; 빈 코어 비편집.

## 비목표 (YAGNI)
- 케이블 사이드패널 편집. 인라인 검증(번호 유일성 등 — 별도). 컬럼 정렬/필터. date/textarea/status 의 EditableField 통합(인스펙터 고유 유지).

## 단계 분해(플랜 예고)
1. 공유 `EditableField`(text/select, 호버 ✎, Enter 커밋, Esc 취소) + 단위테스트.
2. fiberRegisterDescriptor 편집셀 → EditableField 교체(동작 보존).
3. powerRegisterDescriptor 편집 추가(번호/용량/SW) + commitMeta + 테스트.
4. 인스펙터 `Field` → EditableField 채택(동작 보존).
5. 검증 + 최종 리뷰.
