# 선번장 인라인 연결 편집 (자국설비/대국설비 드롭다운) Design

**작성일:** 2026-06-15
**상태:** 설계(사용자 승인) → 사용자 spec 리뷰 → 구현 플랜
**큰 그림:** 광 코어 연결을 선번장(fiber register)에서 직접 편집. 포트 GUI(P2b, 머지됨)는 상태 표시, **연결 행위는 선번장 드롭다운**으로. 후속: 경로(pathPoints) 수동 그리기는 별도.

## 배경 / 문제

선번장(`FiberRegisterView` → `ConnectionRegisterGrid` + `fiberRegisterDescriptor`)은 OFD-SLOT 별 코어 1..N 을 표로 보여준다. 현재 `근접자산`·`상대국측` 컬럼은 **읽기전용**(설비명 또는 —)이고, 코어를 연결하려면 도면에서 케이블을 그려야 한다. 사용자는 **선번장에서 바로** 자국·대국 설비를 골라 연결하길 원한다.

정찰 결론(2026-06-15):
- OUT 코어 케이블(`fiberWrite.buildCoreOutCable`, `fiberWrite.ts:49-60`)은 `sourceAssetId=설비, targetAssetId=슬롯, targetRole='OUT', number=코어번호, specParams={}, pathPoints=null` 구조. **도면 경로(`pathPoints`)는 nullable**.
- 렌더러(`connectionRenderer.ts:37-43`)는 `pathPoints` 없으면 **두 끝점 직선 자동 렌더**. → 선번장에서 만든 연결은 도면·현황 연결 그리드에 **직선으로 자동 노출**(추가 작업 불필요).
- 워킹카피·커밋은 **변전소 전역**(`substationStore.ts:42-48`, 전역 `POST /commit`). → **대국(다른 변전소) 케이블 쓰기 가능**. 대국 설비도 전역 slim 피드로 클라이언트 열거 가능.
- 끝점은 `patch('cables', id, {sourceAssetId})`로 **제자리 수정** 가능(메타 보존).

## 결정 사항(사용자 확정)

1. **대국설비 = 실제 대국 케이블 (A안, cross-substation).** 로컬 선번장에서 대국설비를 고르면 **대국 변전소의 OUT 코어 케이블이 실제 생성/수정**된다(단일 진실원천). 로컬 편집이 대국 변전소 데이터를 staged → 커밋 시 함께 반영. (denormalized 주석 B안 기각.)
2. **드롭다운 후보 = 해당 변전소에 배치된 모든 설비**(OFD·광슬롯 등 **구조/통로 자산 제외**). 광단말이 랙 모듈이어도 선택 가능.
3. **도면 연결 = 직선 자동**(pathPoints null). 경로 수동 그리기는 비범위.

## 목표

선번장에서 코어별로 **자국설비·대국설비를 인라인 드롭다운으로 연결/변경/해제**:
- 컬럼 워딩 `근접자산`→**자국설비**, `상대국측`→**대국설비**.
- 자국설비 선택 → 자국 OUT 케이블(설비↔자국슬롯, number=i) upsert.
- 대국설비 선택 → 대국 OUT 케이블(설비↔대국 twin 슬롯, number=i) upsert (**대국 변전소에 실제 생성**).
- 자국·대국 **독립**(편도/양측). 변경=endpoint 제자리 patch. `—`=연결 해제(삭제).

**비목표(YAGNI):** 경로(pathPoints) 수동 그리기. 포트 GUI에서의 연결(연결은 선번장에서만). 코어 메타(용도/융착) 편집은 기존 그대로. 새 설비 생성(기존 배치 설비만 선택).

## 아키텍처

### A. 컬럼 워딩 + 인라인 드롭다운 (`fiberRegisterDescriptor.tsx`)
- `근접자산`/`상대국측` 라벨 → `자국설비`/`대국설비`.
- 두 컬럼의 `cell` 을 읽기전용 텍스트 → **전용 셀 컴포넌트**(아래 B)로 교체. sortKey 는 설비명 유지.

### B. `EquipmentSelectCell` (신규, hooks 보유 셀 컴포넌트)
descriptor 는 순수(hooks 불가)이므로, 드롭다운 셀은 자체 컴포넌트로 분리한다.
```tsx
<EquipmentSelectCell side="local" | "remote" coreNumber slot graph />
```
- **후보 목록:**
  - `side="local"` → `useEffectiveAssets()` 중 `substationId === slot.substationId` 이고 **배치(placementKind 있음)** + **구조/통로 자산 제외**(`assetType.connectionKind == null` 즉 distributor/conduit 아님, OFD-SLOT/OFD 제외).
  - `side="remote"` → 같은 필터를 **대국 변전소**(`remoteSlotSubstation(slot.id, graph)` 의 변전소 = twin 슬롯의 substationId)에 적용. twin 슬롯 없으면 비활성.
- **현재값:** 그 코어의 자국/대국 OUT 케이블 endpoint 설비(없으면 `—`).
- **변경 시(onChange):**
  - 빈→설비: `buildCoreOutCable` 로 OUT 케이블 생성(자국=자국슬롯 타겟, 대국=twin 슬롯 타겟, number=coreNumber, 해당 설비 substation) → `put('cables', cable)`.
  - 설비→다른 설비: 기존 OUT 케이블 `patch('cables', id, { sourceAssetId: newEquipmentId })`.
  - 설비→`—`: 확인 후 `remove('cables', id)`.
- 드롭다운 UI 는 기존 `EditableField type="select"` 패턴 재사용(가능하면) 또는 동등한 접근성(select).

### C. 쓰기 헬퍼 (`fiberWrite.ts` 재사용/보강)
- 생성은 기존 `buildCoreOutCable({ equipmentId, slotId, coreNumber, category, ids, substationId })` 재사용. 필요한 fiber-core `CableCategory`(예: CBL-FIBER/코어 카테고리)는 `useCableCategories()` 에서 코드로 조회.
- 자국/대국 공통 — 차이는 slotId(자국슬롯 vs twin슬롯)와 설비/substation 뿐. 한 헬퍼로 양쪽.
- twin 슬롯 id = OPGW(IN-IN)의 반대편 endpoint(기존 `OfdSlotRail`/`slotPorts` 패턴 재사용).

### D. cross-substation 커밋
- 대국 OUT 케이블은 `sourceAssetId=대국설비(대국 substationId)`, `targetAssetId=twin슬롯(대국 substationId)`. 전역 워킹카피에 stage → 전역 `POST /commit` 이 변전소 무관 수용(정찰 확인). 추가 작업 없음.

### E. 도면/현황 — 무변경
- pathPoints 없는 OUT 케이블 → 직선 자동 렌더(`connectionRenderer`), 현황 연결 그리드는 양 끝점 resolve 되면 노출. 선번장 연결이 그대로 보임.

## 데이터 흐름
```
선번장 행(코어 i) → 자국설비 select → buildCoreOutCable(자국슬롯) → put(cables) → overlay → 직선 도면 + 현황
                  → 대국설비 select → buildCoreOutCable(twin슬롯, 대국 substationId) → put(cables) → (대국 변전소 staged)
변경 → patch(cables, id, {sourceAssetId}) ; 해제 → remove(cables, id)
포트 GUI(buildSlotPorts) 는 동일 OUT 케이블에서 상태 재파생 → 색 자동 일치
```

## 엣지/에러 처리
- **twin 슬롯 없음(OPGW 결손):** 대국설비 드롭다운 비활성 + 안내.
- **fiber-core 카테고리 로딩/누락:** 드롭다운 비활성(연결 불가) + 안내.
- **같은 설비 여러 코어:** 허용(광단말 N코어).
- **해제:** 확인 다이얼로그(연결+메타 삭제 경고).
- **빈 행 전체선택 버그:** 이미 수정됨(빈 행 onRowClick null) — 본 작업과 무관, 유지.
- **대국 편집 자각:** 대국설비 변경은 대국 변전소 데이터를 바꾼다 — 커밋 전 overlay 에 보이며, 단일 커밋으로 양쪽 반영(전역 UoW 의도된 동작).

## 테스트 전략
- **쓰기 헬퍼(단위):** 자국 set→OUT(자국슬롯,number,자국 substation), 대국 set→OUT(twin슬롯,대국 substation), 변경→`patch(sourceAssetId)`, 해제→`remove`.
- **EquipmentSelectCell(컴포넌트):** 후보 필터(변전소별·구조자산 제외), 현재값 표시, 선택→put/patch/remove 호출, twin 없음→비활성.
- **descriptor:** 컬럼 라벨 자국설비/대국설비, 셀이 EquipmentSelectCell 렌더.
- **빌드/타입:** 프론트 tsc + build + 전체 vitest green.
- **수동 스모크:** 선번장에서 자국·대국 설비 연결 → 도면 직선·현황 노출·포트 GUI 색 변화·대국 변전소 선번장에 반영(커밋 후).

## 비목표 (YAGNI)
경로(pathPoints) 수동 그리기. 포트 GUI 연결 진입(연결은 선번장). 신규 설비 생성. 코어 메타 편집 변경.

## 단계 분해(플랜 예고)
1. fiber OUT-코어 쓰기 헬퍼(생성/patch/삭제) 정리 + 단위테스트(자국/대국/twin).
2. `EquipmentSelectCell`(후보 필터·현재값·커밋) + 테스트.
3. `fiberRegisterDescriptor` 워딩 변경 + 자국/대국 컬럼을 셀 컴포넌트로 교체 + 테스트.
4. 검증(전체 vitest/tsc/build) + 최종 리뷰 + 수동 스모크.
