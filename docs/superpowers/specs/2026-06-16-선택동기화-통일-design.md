# 선택 동기화 통일 + 포트 케이블 진입점 Design

**작성일:** 2026-06-16
**상태:** 설계(사용자 승인: 단일 selectedCore 소스 + 파생 하이라이트, 컴포넌트 core 있으면 동기화) → spec 리뷰 → 구현 플랜
**큰 그림:** 그리드뷰·사이드패널(포트/CB)·연결탭의 선택을 **`selectionStore (selectedAssetId, selectedCore)` 단일 소스**로 통일하고, 캔버스/토폴로지 하이라이트는 그로부터 **파생(중앙 단일 effect)** 한다. 파편화된 per-panel `startTrace` 와 그리드의 `tracingCableId` 이중 하이라이트를 제거해 **모든 경우 일관 동작**. 이후 경로슬롯 포트뷰에 선택 포트 출발 케이블 진입점(자국만)을 추가.

---

## 1. 배경 / 현재 문제 (정찰 확정)

- **선택 소스 2개(파편화):**
  - `selectionStore`(`selectedAssetId`,`selectedCore`) — 그리드 행 클릭(`setSelected`), 포트 클릭(`setSelectedCore`), 피더 CB 클릭(`setSelectedCore`).
  - `pathHighlightStore.tracingCableId` — `startTrace`/`highlightDiagram` 가 세팅. **그리드가 이것도 하이라이트 기준으로 읽음**(`ConnectionRegisterGrid.RowView` line 140-142).
- **이중 선택 버그:** 포트 클릭(`selectedCore=N` → N행 하이라이트) + 연결탭 클릭(`highlightDiagram` → `tracingCableId` → 그 행 하이라이트) → **두 행 동시 하이라이트**.
- **연결탭 비동기화:** `AssetConnectionsTab` 클릭이 `highlightDiagram(seedCableId,…)` 로 `tracingCableId` 만 세팅, `selectedCore` 미경유 → 그리드/포트와 안 맞음.
- **per-panel 하이라이트 effect(파편화):** `SlotPortsPanel`·`FeederCircuitsPanel` 각각 `selectedCore` → `startTrace` effect 보유. 탭 전환 시 포트뷰('정보' 탭)와 연결탭('연결' 탭)은 **공존 안 함**(`AssetInspector` 별도 탭) → 연결탭에선 SlotPortsPanel effect 미실행 → 하이라이트 파생이 탭마다 갈림.
- **재사용 구조:** `ConnectionRegisterGrid` + descriptor(`rowCore`/`rowTraceCableId`/`onRowClick`), `buildConnectionDiagram`(연결탭 컴포넌트), `startCableConnection`(케이블 진입), `useTraceGraph`.

## 2. 목표 / 비목표

**목표**
- 그리드·포트패널·CB레일·연결탭이 **단일 `selectedCore`** 로 읽고/쓰며 어디서 클릭하든 동일 동기화.
- 하이라이트(`tracingCableId`/캔버스/토폴로지)는 selectedCore 에서 **중앙 단일 effect** 로 파생.
- 이중 선택 버그 제거. per-panel `startTrace`·그리드 `tracingCableId` 분기·`rowTraceCableId` **데드코드 제거**.
- 경로슬롯 포트뷰에 **선택 포트 출발 케이블 진입점(자국 OUT만)** 추가.

**비목표(YAGNI)**
- 피더 연결탭을 CB별로 분해(승인안: 분배 전체 = core null = 자산만 선택).
- `tracingCableId` 자체 제거(캔버스·토폴로지·`AssetConnectionsSection`·equipment `ConnectionDiagram` 등 다른 소비처는 그대로 — 단, **그리드는 더 이상 안 읽음**).
- 백엔드 변경.

## 3. 단일 규칙

> **선택 = `(selectedAssetId, selectedCore)` 하나.** 모든 뷰가 이걸 읽어 active 표시, 클릭 시 이것만 기록.
> **하이라이트 = 선택에서 파생.** `(selectedAssetId, selectedCore)` 변화 → `resolveSelectedCable` 로 케이블 1개 → `startTrace`(없으면 `clearHighlight`). 중앙 1곳.
> **연결탭 항목 ↔ selectedCore:** 컴포넌트가 sub-선택(core)을 가지면 그 core 로, 없으면 자산만.

## 4. 컴포넌트별 변경

### 4.1 `resolveSelectedCable` + 중앙 하이라이트 훅 (신규)
- `frontend/src/features/workspace/selectionHighlight.ts`:
  ```ts
  // (asset, core) → 하이라이트 seed 케이블. core 센티넬 입력(0=FEEDER_INPUT_CORE)=IN 케이블,
  // 그 외 core=그 자산에 닿는 number===core 케이블(슬롯 OUT코어/피더 OUT분기 통일).
  export function resolveSelectedCable(assetId: string | null, core: number | null, cables): string | null
  // (selectedAssetId, selectedCore) 변화 → resolveSelectedCable → startTrace / clearHighlight.
  export function useSelectionHighlight(): void
  ```
- `resolveSelectedCable(assetId, core, cables)`:
  - `core == null || !assetId` → null.
  - `touching = cables.filter(c => c.sourceAssetId===assetId || c.targetAssetId===assetId)`.
  - `core === FEEDER_INPUT_CORE(0)` → `touching.find(c => roleAt(c,assetId)==='IN')?.id ?? null`.
  - else → `touching.find(c => (c.number ?? null) === core)?.id ?? null`.
- `useSelectionHighlight`: `useEffect([selectedAssetId, selectedCore, graph])` → `const id = resolveSelectedCable(...)` → `id ? startTrace(id) : clearHighlight()`. `WorkspacePage` 에서 1회 호출(항상 마운트).
- **대체:** `SlotPortsPanel`·`FeederCircuitsPanel` 의 `selectedCore→startTrace` effect 제거.

### 4.2 그리드 `ConnectionRegisterGrid.RowView`
- highlight = `(!!selectedId && selectedAssetId===selectedId && (rowCore ?? null)===(selectedCore ?? null))` **만**. `tracingCableId===traceCableId` 분기 + `tracingCableId`/`traceCableId`/`rowTraceCableId` prop **삭제**.
- descriptor 의 `rowTraceCableId`(fiber/power) 및 타입의 해당 필드 제거(데드코드).

### 4.3 연결탭 `AssetConnectionsTab` + `buildConnectionDiagram`
- `DiagramComponent` 에 `core: number | null` 추가. `buildConnectionDiagram` 에서 각 회로마다:
  `회로 케이블 중 self(=isSelf) 에 닿고 number!=null 인 것들의 distinct number 가 정확히 1개면 그 값, 아니면 null`.
  (슬롯 회로=OUT 코어 1개 → core=코어번호. 피더 분배=분기 여러 number → null. 단일분기 피더 → 그 CB.)
- `AssetConnectionsTab`:
  - 행 클릭 → `useSelectionStore.getState().setSelected(assetId, comp.core)` (코어 있으면 코어까지, 없으면 자산만). `highlightDiagram` 호출 제거.
  - active = `selectedAssetId===assetId && selectedCore===comp.core` (`useSelectionStore` 구독). `tracingCableId` 비교 제거.
  - `[상세]`(↗) = `startTrace(comp.seedCableId); openTopology();` 유지(명시적 토폴로지).
  - 언마운트 `clearHighlight` 제거(중앙 훅이 선택 비움으로 처리; 탭 전환에 하이라이트 유지).

### 4.4 포트/CB 패널
- `SlotPortsPanel`: `selectedCore→startTrace` effect + 언마운트 clear 제거. 포트 클릭 = `setSelectedCore` 유지.
- `FeederCircuitsPanel`: `selectedCb→startTrace` effect + 언마운트 clear 제거. CB/입력 클릭 = `setSelectedCore` 유지.

## 5. 포트뷰 케이블 진입점 (자국 OUT)
- `SlotPortsPanel`: 선택 포트가 **자국 free(`localCableId` 없음)** 일 때 상세카드에 **"케이블 연결"** 버튼.
  - `startCableConnection({ source: { containerAssetId: anchorRect.anchorId, position: anchorRect.position, slotId: slot.id, coreNumber: selectedCore, role: 'OUT' } })` 후 `nav.gotoAsset(slot.id)`(평면도 진입, 출발지 자동 주입) — 피더 `addCb` 와 동일 패턴.
  - **자국만**: `role:'OUT'`(슬롯 자국 측 → 로컬 단말). 대국 OPGW(IN-IN)는 이 진입점 비대상.
  - 점유 포트(localCableId 있음)면 버튼 숨김(빈 포트에만 출발 — 1대다 자산 규칙).

## 6. 동기화 결과 (모든 경우 일관)
코어 N 선택(어디서 클릭하든) → 그리드 N행 + 포트패널 포트N + 연결탭 코어N 항목 + 캔버스 하이라이트 = 동시·일관. 자산만 선택(피더 분배 등 core=null) → 자산 행/패널만, 특정 코어 없음.

## 7. 엣지 / 에러
- 코어에 케이블 없음(빈 포트) → `resolveSelectedCable` null → `clearHighlight`(하이라이트 없음, 선택은 유지).
- 연결탭 컴포넌트 core 모호(피더 다분기) → null → 자산만.
- 선택 자산이 현재 그래프에 없음 → resolve null → clear.
- 진입점: 슬롯 anchor/position 미해소 시 버튼 비활성.

## 8. 테스트 전략
- **`resolveSelectedCable`(단위):** 슬롯 OUT#코어 / 피더 OUT#CB / 피더 입력(0→IN) / 코어 null·없음 → 케이블/널.
- **`buildConnectionDiagram` core(단위):** 슬롯 회로 → core=코어번호, 피더 다분기 → null, 단일분기 → CB.
- **`AssetConnectionsTab`(컴포넌트):** 행 클릭 → `setSelected(asset, core)`(highlightDiagram 미호출), active=selectedCore 기준, [상세]→startTrace+openTopology.
- **그리드 RowView(컴포넌트/기존):** selectedCore 만으로 하이라이트(tracingCableId 무관) — 이중 선택 회귀 방지.
- **`SlotPortsPanel`(컴포넌트):** 자국 free 포트 선택 → 진입점 버튼 → `startCableConnection` 호출(source role OUT/slotId/coreNumber). 점유 포트 → 버튼 없음.
- **빌드/타입:** `npx tsc` + 전체 `npx vitest` green(frontend).
- **수동 스모크:** 선번장에서 포트 클릭→연결탭/그리드 동시 활성, 연결탭 클릭→포트/그리드 동시 활성(이중선택 없음). 포트뷰 진입점→평면도 출발 주입.

## 9. 파일 영향
- 신규: `workspace/selectionHighlight.ts`(+테스트).
- 수정: `connections/registerGrid/ConnectionRegisterGrid.tsx`(RowView·prop), `connections/registerGrid/registerTypes.ts`(rowTraceCableId 제거), `fiber/fiberRegisterDescriptor.tsx`·`power/powerRegisterDescriptor.tsx`(rowTraceCableId 제거), `connections/connectionDiagram.ts`(component.core), `connections/components/AssetConnectionsTab.tsx`(selectedCore 동기화), `fiber/components/SlotPortsPanel.tsx`(effect 제거 + 진입점), `power/components/FeederCircuitsPanel.tsx`(effect 제거), `pages/WorkspacePage.tsx`(useSelectionHighlight 마운트).
- 재사용(불변): `pathHighlightStore.startTrace`/`clearHighlight`/`openTopology`, `startCableConnection`, `NetworkTopologyModal`, `useTraceGraph`.

## 10. 핵심 불변식
- 선택 = `selectionStore` 하나. 하이라이트는 파생(중앙 1곳).
- 그리드는 `tracingCableId` 를 **안 읽는다**(selectedCore 만).
- 종류별 조건분기 없음 — core 유무는 데이터(컴포넌트 number)에서.
