# 그리드 디자인·정렬·선택 통일 (현황 ↔ 케이블 그리드) Design

**작성일:** 2026-06-15
**상태:** 설계 확정(사용자 승인) → 구현 플랜 대기

## 배경

같은 "그리드 뷰"인데 현황 그리드(`NodeStatusView`)와 케이블 대장 그리드(`ConnectionRegisterGrid` — 선번장/계통)의 룩·기능이 따로 논다. 케이블 그리드는 이미 테이블 크롬(`<table w-full border-collapse>`, `h-9` 행, `text-[13px]` 셀, 선택행 스타일 클래스)이 현황과 **거의 동일**하나, 세 가지가 빠져 있다:

1. **선택행(눌림) 표시가 "선택" 기준이 아님.** 케이블 그리드 `RowView` 는 행 강조를 `tracingCableId`(경로 트레이스)로만 한다. 현황은 `selectedAssetId`(행 클릭→선택)로 눌림 표시. → 케이블 그리드에서 클릭한 행이 눌려 보이지 않는다.
2. **컬럼 정렬 없음.** 현황은 헤더 클릭 → 3단계 정렬(asc→desc→해제) + Chevron 아이콘 + 활성 헤더 스타일. 케이블 그리드 헤더는 정적 텍스트.
3. **헤더 크롬 불일치.** 현황 헤더는 버튼(정렬)·활성 스타일(`bg-surface-3`·`font-semibold`)·스마트 패딩(첫 컬럼 `pl-4`, 끝 컬럼 `pr-4`). 케이블은 정적 `px-2`.

현황의 정렬 비교(`compareBy`: text(ko-locale)/date/status, 빈값 뒤로)·헤더 버튼·선택행 스타일은 **NodeStatusView 안에 bespoke**로 박혀 있다. 복제하면 드리프트 위험.

## 목표

현황의 정렬·헤더·선택행 로직을 **공유 모듈로 추출**하고, **현황·케이블 그리드 양쪽이 동일하게 사용**한다(단일 소스). 케이블 그리드는 섹션 구조를 유지하되 테이블 크롬·정렬·선택행을 현황과 일치시킨다. **기능은 그리드마다 다를 수 있어도(예: 케이블 편집셀) 룩·정렬·선택 UX는 동일.**

**비목표:** 케이블 그리드에 검색/필터 칩(현황의 검색·상태필터·종류칩 — 케이블은 섹션 구조라 불필요, YAGNI). 행 가상화. 다중 컬럼 정렬. 케이블 그리드 섹션 평탄화(섹션 유지).

## 아키텍처

### 공유 그리드 모듈 (`frontend/src/components/grid/`)
- **`compareBy.ts`** — `compareBy(type: 'text'|'date'|'number'|'status', a: string|number|null, b: string|number|null): number`. text=ko localeCompare, date=Date getTime 차, number=수치, status=문자비교. **빈값(null/'')은 방향 무관 항상 뒤로.** (NodeStatusView 에서 추출.)
- **`useGridSort.ts`** — `useGridSort()` → `{ sort: {col,dir}|null, cycleSort(col): void }`. cycleSort: 없거나 다른 col → asc / asc → desc / desc → null(해제). + 순수 `sortRows<Row>(rows, accessor, type, dir)`(stable: 동값은 원순서 유지, 빈값 뒤로).
- **`SortableHeaderCell.tsx`** — `<SortableHeaderCell label active dir onClick width? first? last? />`. 현황 헤더 버튼 패턴: `<th aria-sort><button onClick … class="group … hover:bg-surface-3 …">label + Chevron(Up/Down/UpDown)</button></th>`. 정렬 불가 컬럼용 정적 변형(버튼 없이 같은 패딩/룩)도 제공(또는 `onClick` 없으면 정적).
- **`rowClasses.ts`** — `SELECTED_ROW = 'bg-info-bg shadow-[inset_3px_0_0_var(--primary)]'`, `ROW_BASE = 'h-9 cursor-pointer border-b border-line transition-colors'`, `ROW_HOVER = 'hover:bg-surface-2 active:bg-surface-3'`. (현재 양쪽이 동일 문자열을 인라인으로 — 상수화.)

### ConnectionRegisterGrid 변경
- **정렬 상태 1개**(그리드 단위, `useGridSort`) → **각 섹션 rows 에 동일 적용**. 한 컬럼 헤더 클릭 → 모든 섹션이 그 컬럼으로 정렬(섹션마다 독립 thead 라 헤더는 N개지만 sort 상태는 공유 → 일관). 정렬키 없는 컬럼은 정적 헤더.
- **`SortableHeaderCell`** 로 헤더 렌더(첫/끝 컬럼 스마트 패딩). 컬럼에 `sortKey` 있으면 버튼+Chevron, 없으면 정적.
- **`RowView` 선택행**: `useSelectionStore(s=>s.selectedAssetId)` 구독 → `selected = rowSelectedId != null && selectedAssetId === rowSelectedId`. 강조 = `selected || tracing`(둘 다 동일 `SELECTED_ROW` 스타일). `rowSelectedId` = `descriptor.onRowClick(row, child)`(SectionView 가 계산해 전달).
- **`RegisterColumn`** 확장: `sortKey?(row): string|number|null`, `sortType?: 'text'|'date'|'number'|'status'`(기본 'text'). 렌더-함수 `cell` 은 그대로.

### descriptor 정렬키
- **fiber**: 코어(number, coreNumber), 근접자산(text, nameById 이름), 상대국측(text, farName), 용도/수용내역(text), 융착/사용(text). 기본 정렬 없음(buildSlotCoreRows 의 1..N 원순서).
- **power**: 번호(text 또는 number, cbNumber), 부하(text, loadName), 용량(text, capacity), 규격(text, spec), SW(text, switchState).

### NodeStatusView 채택
자기 `compareBy`·헤더 버튼(Chevron)·선택행 스타일을 공유 모듈로 교체(`useGridSort`/`compareBy`/`SortableHeaderCell`/rowClasses). COLUMN_DEFS(accessor/type)·필터·검색은 현황 고유로 유지. `NodeStatusView.test` 로 정렬·선택·필터 동작 보존.

## 데이터 흐름
```
[정렬]  헤더 클릭 → useGridSort.cycleSort(col) → sort 상태
        → 각 섹션 rows = sortRows(rows, col.sortKey, col.sortType, dir)  (sort 없으면 원순서)
[선택]  행 클릭 → onRowClick → setSelectedAssetId(id)  [기존]
        → RowView: selectedAssetId===rowSelectedId → SELECTED_ROW 눌림 [신규]
[트레이스] tracingCableId===row → SELECTED_ROW (공존)  [기존]
```

## 엣지/에러 처리
- **정렬 해제(3rd click)** → 원순서 복원(stable, 원 인덱스).
- **빈값**: 정렬 시 항상 뒤로(방향 무관). 빈 코어 행(용도 등 빈)도 동일.
- **선택행 충돌**: 빈 코어들이 onRowClick 으로 모두 OFD id 로 해소 → 빈 코어 선택 시 그 OFD 의 빈 코어들이 함께 눌려 보일 수 있음(점유 행은 고유 설비라 정확). 마이너 — 필요 시 descriptor 가 행-고유 selectedId 제공(현재는 onRowClick 재사용). **점유 행 선택은 정확.**
- **섹션별 헤더 N개**: 정렬 상태 공유라 모든 섹션 헤더가 같은 활성 표시(일관).
- **정렬키 없는 컬럼**: 정적 헤더(클릭 무동작) — 현황과 달리 케이블은 일부만 정렬 가능.

## 테스트 전략
- **공유 순수 단위**: `compareBy`(text ko·date·number·status·빈값뒤로), `sortRows`(stable·dir·빈값), `useGridSort.cycleSort`(3단계).
- **SortableHeaderCell**: 활성/비활성 아이콘·클릭 콜백·aria-sort.
- **ConnectionRegisterGrid**: 헤더 클릭→섹션 rows 정렬, 행 클릭→selectedAssetId→선택행 스타일(클래스 단언), 정렬키 없는 컬럼 정적. 기존 `FiberRegisterView.test`/`PanelCircuitView.test` 보존.
- **NodeStatusView**: 공유 모듈 채택 후 `NodeStatusView.test` 정렬/선택/필터 보존.
- **빌드/타입**: 프론트 tsc + build + 전체 vitest green.
- **수동 스모크**: 현황·선번장·계통 그리드가 시각·정렬(asc/desc/해제·Chevron)·선택행(클릭 눌림) 동일.

## 비목표 (YAGNI)
- 케이블 그리드 검색/필터/종류칩. 다중정렬. 정렬 영속(새로고침 유지). 행 가상화. sticky 헤더(섹션 구조라 케이블은 비sticky 유지).

## 단계 분해(플랜 예고)
1. 공유 그리드 모듈(compareBy·useGridSort·sortRows·SortableHeaderCell·rowClasses) + 단위테스트.
2. ConnectionRegisterGrid: SortableHeaderCell 헤더 + 섹션별 정렬 + RowView selectedAssetId 선택행 + RegisterColumn.sortKey.
3. fiber/power descriptor 에 sortKey 부여.
4. NodeStatusView 공유 모듈 채택(동작 보존).
5. 검증 + 최종 리뷰.
