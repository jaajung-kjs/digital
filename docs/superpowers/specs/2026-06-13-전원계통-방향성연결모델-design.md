# 전원계통 + 방향성 연결 모델 (1대다 자산 통일) 설계

**작성일:** 2026-06-13
**범위:** 전원계통 슬라이스(distributor) + 그 토대인 방향성 연결 모델. conduit(광/FiberPath 흡수)·접지는 후속.

---

## 1. 목표

원주가 엑셀로 관리하던 **전원계통(분전반 CB 회선 + 충전기/UPS)**을 SSOT 위 **계통 그리드 뷰**로 넣는다. 그리고 그 과정에서, 선번장에서 시작한 "허브+슬롯" 패턴을 **단일 방향성 연결 모델**로 일반화해 `root`·`BRANCH` 같은 예외를 제거한다. 현황·선번장·**계통** 세 그리드 + SSOT로 이어진 평면도 — 이 골격을 완성한다.

## 2. 북극성 — 단일 방향성 연결 모델 (대화로 확정)

**새 자산 모델/테이블 없음. 기존 자산DB(Asset + parentAssetId + Cable + attributes) 그대로 + 두 조각만 추가.**

- **모든 것은 자산.** 자산이 자산을 품고(`parentAssetId`), 자산끼리 케이블로 잇는다. "1대다"는 타입이 아니라 *모양* — 중첩(분전반)·독립(UPS) 자유.
- **추가 ① `AssetType.connectionKind`**: `none | distributor | conduit` (행동은 *종류*가 선언).
  - **distributor** = fan-out(입력 1 → 출력 N, 짝 없음): 피더·충전기·UPS.
  - **conduit** = 번호 짝 통과(채널 i ↔ i): OFD. *(이번 슬라이스 트레이서는 conduit 미구현 — 플래그만 심음)*
  - **none** = 수동 끝점: 부하·일반설비·컨테이너(분전반).
- **추가 ② 케이블의 연결별 사실**: distributor엔 **흐름 방향**, conduit엔 **채널 번호**(=기존 `fiberPortNumber`).
- **방향성 트레이서**: 노드의 `connectionKind`로 분기 — distributor는 입력↔출력 흐름을 따라 상류로, **입력 없는 distributor(=발전원)에서 자연 종료.** → **`rootKindForCableType` 폐기**(AC/DC 한정; FIBER는 후속까지 기존 유지).
- **`BRANCH`(분기) 자산 삭제**: 1 CB = 1 부하 회선 = **출력 케이블 한 개**. CB 메타는 케이블 속성.
- **세 그리드(현황·선번장·계통) = "1대다 자산의 연결 레지스트리"**(선번장의 FiberCore 역할을 일반화). 평면도 = 유일 시각화(하이라이트).

## 3. 데이터 모델

### 3.1 `AssetType.connectionKind` (신규 컬럼)
```prisma
// AssetType 에 추가
connectionKind String? @map("connection_kind") @db.VarChar(20) // 'distributor'|'conduit'|null(=none)
```
시드(`assetTypes.ts`):
- `FEEDER`(피더) → **distributor**
- `CHARGER`(충전기) · `UPS` → **distributor**
- `OFD` → **conduit** *(후속 트레이서용. 지금은 표식만)*
- `DIST`(분전반) → null *(컨테이너; 자기 전기역할 없음, 피더를 품음)*
- 나머지(부하·BATTERY 등) → null
- `BRANCH` → **제거**(시드에서 삭제, 아래 마이그레이션 주의 참조)

### 3.2 케이블 IN/OUT 역할 (신규 필드)
케이블을 **방향성 자산에 꽂을 때 그 끝을 IN(입력)/OUT(출력)으로 지정**한다(우리가 합의한 표현).
```prisma
// Cable 에 추가 — 방향성 자산에 닿는 끝의 역할
sourceRole String? @map("source_role") // 'IN'|'OUT'|null  (source 끝이 방향성 자산일 때만)
targetRole String? @map("target_role") // 'IN'|'OUT'|null  (target 끝이 방향성 자산일 때만)
```
- distributor 자산 D 기준: **입력 케이블** = D 쪽 역할이 `IN` · **출력 케이블** = D 쪽 역할이 `OUT`.
- 일반(none) 끝은 역할 `null`. 양끝 모두 distributor면(예: 충전기 `OUT` → 분전반 `IN`) **양쪽 역할을 각각 지정**(서로 반대).
- ⚠️ `source/target`은 그리는 *순서*일 뿐 방향 의미 없음 → 방향은 이 IN/OUT 역할로만.

### 3.3 CB(회선) 메타 = 출력 케이블 속성
CB의 `번호·용량·SW상태`는 출력 케이블의 **`specParams`(기존 free-form JSON)** 에:
```jsonc
specParams: { "cbNumber": 7, "capacity": "30A", "switchState": "ON" }
```
- `전원케이블 규격`(FR-14sq 등) = 케이블 `categoryId`(기존 CableCategory) — **새로 저장 안 함**.
- `용도(부하)` = 케이블이 닿는 부하 자산 → **도출**(저장 안 함).
- → CB 한 줄 = 케이블 1개 + specParams 3필드. **새 테이블 0**(FiberCore도 후속엔 이 자리로 수렴).

### 3.4 마이그레이션 주의
- `connectionKind`·`sourceRole`·`targetRole` 컬럼 추가(모두 nullable, 무해).
- `BRANCH` 자산 타입 제거: 기존 데이터에 BRANCH 인스턴스가 있으면 출력 케이블로 이관하거나, 없으면 시드에서만 제거. **운영 데이터 확인 후 결정**(현재 시드 단계라 인스턴스 0일 가능성 높음).

## 4. 방향성 트레이서

`connectionKind`-aware 상류 추적(전원):
```
상류추적(자산 A):
  A 가 distributor 면 → A 의 입력 케이블(A 쪽 역할=IN)을 따라 반대편으로
  A 가 none/부하면  → 닿은 케이블의 상대편으로
  도달 노드가 distributor 면 다시 그 입력 케이블(역할=IN)로 계속
  입력 케이블이 없는 distributor(=발전원)에서 종료
```
- **`rootKindForCableType`의 AC/DC stop 제거** — 흐름이 발전원에서 자연 종료.
- **FIBER는 이번 슬라이스에서 기존 경로(tracePathToRoot/usePortStatus) 유지** — conduit 트레이서는 후속.
- 트레이서는 `connectionKind`로 분기: distributor=흐름추적 / conduit=후속 / none=통과.

## 5. 뷰 — 계통 그리드 (선번장과 동형)

### 5.1 구조
- top-level 뷰에 **"계통"** 추가 (`WorkspacePage.VIEWS`: 현황·평면도·선번장·**계통**).
- 변전소의 distributor 자산을 모아 렌더:
  - **분전반(컨테이너)** → 그 안 **피더 섹션**(상용/DC/UPS) → **CB(출력 케이블) 행**
  - **독립 distributor(충전기·UPS)** → 자기 출력 행
```
▸ 분전반 #1
  상용전원반 (피더)              사용 5/8
   CB  부하           용량   케이블     SW
   1   #1충전기 AC    50A    FR-14sq    ON
   2   —             50A    —          OFF
  DC 48V (피더)                 사용 6/12
   ...
```
- 컬럼: `CB번호 · 부하(도출) · 용량 · 케이블규격(category) · SW상태`. 빈 CB도 표시(빈 회선).
- **현황 그리드 통일 양식 그대로**(h-9 행·text-[13px]·헤더 bg-surface-2 — 선번장 retrofit과 동일 규약).

### 5.2 상호작용 — 세 그리드 일관(★ 선번장 retrofit 포함)
```
그리드 행 클릭 → 사이드패널(연결탭) 열림  (하이라이트 X)
   연결탭에서 연결 클릭 → 그때 평면도 하이라이트 이동
```
- 행 클릭 = `setSelectedAssetId(부하 자산)` 만. **즉시 trace 안 함.**
- 하이라이트는 연결탭(`AssetConnectionsSection`, 이미 행클릭→trace 보유)이 담당.
- **선번장 retrofit**: `OfdFiberRegister` 행 onClick 에서 `startTrace` 제거(=`setSelectedAssetId`만). 두 그리드 동작 통일.

### 5.3 드로잉 UI
- distributor 자산에 케이블을 꽂을 때 **"입력 / 출력" 지정**(기본 출력, 입력만 표시) → 케이블의 그 끝 역할(`sourceRole`/`targetRole`=IN/OUT) 세팅. 일반 자산끼린 안 물음.

## 6. 범위 밖 (후속, 명시)

- **conduit 트레이서 + 광/FiberPath 흡수**: OFD=conduit, OPGW=트렁크 케이블, 코어번호=케이블 속성, FiberCore→케이블 속성, usePortStatus 재작성. *(전원에서 distributor 증명 후)*
- **선번장·계통 그리드 컴포넌트 완전 일원화**(지금은 동형 별도 컴포넌트, 후속에 제네릭 1개로).
- 충전기/UPS/축전지 인벤토리 fieldTemplate(규격·입출력·감시 등) 정비.
- 접지계통(엑셀에 데이터 없음 — 보류).
- 전체 SLD 자동작도(분전반→충전기→AC메인 그래프 그림) — 방향성 트레이스 위에 후속.

## 7. 재사용 / 신규

**재사용:** Asset/parentAssetId/Cable/specParams/CableCategory, `selectionStore`, `AssetConnectionsSection`(연결탭), `usePathHighlightStore`, 워킹카피 단일 커밋, 현황 그리드 양식, `WorkspacePage.VIEWS`.
**신규/수정:** `AssetType.connectionKind`(+시드) / `Cable.sourceRole·targetRole`(IN/OUT, +드로잉 UI) / CB 메타 specParams / 방향성 트레이서(distributor, rootKind AC·DC 제거) / 계통 그리드 컴포넌트(`PanelCircuitView` 등) + top-level 뷰 / `OfdFiberRegister` 인터랙션 retrofit / `BRANCH` 제거.

## 8. 리뷰 필요 — 기본값으로 잡아둔 결정

1. **CB 메타를 케이블 `specParams` JSON**(cbNumber/capacity/switchState)에 — 타입 컬럼 추가 대신. OK?
2. **케이블 IN/OUT 역할 = `Cable.sourceRole/targetRole`**('IN'|'OUT'|null, 방향성 자산에 닿는 끝에만 지정). OK?
3. **계통 그리드는 선번장과 *동형 별도 컴포넌트*** 로 시작(제네릭 일원화는 후속). OK?
4. **`BRANCH` 제거 범위** — 시드만 제거 vs 기존 인스턴스 이관 필요. (운영 데이터 확인)
5. **선번장 인터랙션 retrofit**(즉시 하이라이트 → 연결탭 클릭 시 하이라이트)을 이 슬라이스에 포함. OK?
6. **FIBER 트레이스는 무변경**(conduit·흡수는 후속) — 이번엔 distributor만. OK?
