# 전원계통 + 방향성 연결 모델 (1대다 자산 통일) 설계

**작성일:** 2026-06-13 (최종 수렴본 — 긴 설계 대화 결과)
**범위:** 전원계통 슬라이스(distributor) + 그 토대인 방향성 연결 모델. conduit(광/FiberPath·FiberCore 흡수)·접지·전체 SLD 자동작도는 후속.

---

## 1. 목표

원주가 엑셀로 관리하던 **전원계통(분전반 CB 회선 + 충전기/UPS)**을 SSOT 위 **계통 그리드 뷰**로 넣는다. 그 과정에서 선번장에서 시작한 패턴을 **단일 방향성 연결 모델**로 일반화해 `root`·`BRANCH`·`FiberCore`(후속) 같은 예외/곁가지를 녹인다. **현황·선번장·계통 세 그리드 + SSOT 평면도** 골격을 완성한다.

## 2. 북극성 — 단일 모델 (대화로 확정)

**새 자산 모델/테이블/노드 없음. 기존 자산DB 그대로 + 두 조각만 추가.**

### 핵심 불변식
- **모든 것은 자산.** 자산이 자산을 품고(`parentAssetId` = *물리적 그룹핑*), 자산끼리 **케이블**로 잇는다(= *전기 흐름*). **중첩 ≠ 흐름**: 흐름은 오직 케이블로 표현.
- **자산도 케이블도 "타입 템플릿 + 속성 가방"**으로 동형:
  | | 타입(필드 정의) | 인스턴스 값 |
  |---|---|---|
  | 자산 | `AssetType.fieldTemplate` | `Asset.attributes` |
  | **케이블** | **`CableCategory.specTemplate`** | **`Cable.specParams`** |
  → CB의 `번호·용량·SW상태`, 광코어의 `용도·수용내역·융착`은 **그 도메인 케이블 카테고리의 specTemplate 필드** = 케이블 속성. 별도 노드·테이블 0.
- **1대다(distributor)** 자산: 입력 1 + 출력 N, 출력은 **케이블**. `AssetType.connectionKind`로 선언.
- **방향**: 케이블의 **IN/OUT 역할**(방향성 자산에 닿는 끝). 방향성 트레이서가 IN 따라 상류로 → **입력 없는 distributor(=발전원)에서 자연 종료 → `root` 폐기.**
- **세 그리드 = "1대다 자산의 케이블-속성 레지스트리"**: 현황(자산)·**선번장**(OFD의 광케이블)·**계통**(분전반/피더의 전원케이블). 평면도 = 유일 시각화(하이라이트).

## 3. 범위: 전원(distributor) 슬라이스

### 3.1 구조 — CB는 케이블, 분기 노드 없음
```
분전반 (컨테이너, connectionKind=none — 물리 그룹핑만)
 └ 피더 (distributor)   ← 상용/DC/UPS 버스
충전기 / UPS (독립 distributor)

흐름(전부 케이블):
  충전기/상용 ──케이블(피더쪽 IN)──> 피더
  피더 ──────케이블(피더쪽 OUT, = CB 회선)──> 부하
```
- **분기(BRANCH) 삭제.** CB = "피더 → 부하" **출력 케이블 한 개**, 차단기 데이터는 그 케이블 속성.
- 분전반↔피더 = **containment**(그리드 그룹핑용), 흐름 아님. 흐름은 충전기→피더→부하 **케이블**.
- 피더·충전기·UPS = distributor(출력=케이블) → IN/OUT 역할로 방향. **흐름 트레이스는 순수 케이블 + IN/OUT 단일 메커니즘**(containment-흐름 예외 없음).

### 3.2 데이터 모델 (추가 3개)
```prisma
// AssetType — 1대다/흐름 선언
connectionKind String? @map("connection_kind") @db.VarChar(20) // 'distributor'|'conduit'|null(=none)

// Cable — 방향성 자산에 닿는 끝의 역할
sourceRole String? @map("source_role") // 'IN'|'OUT'|null
targetRole String? @map("target_role") // 'IN'|'OUT'|null
```
- `connectionKind` 시드: `FEEDER`·`CHARGER`·`UPS` = distributor / `OFD` = conduit(후속용 표식) / `DIST`(분전반)·나머지 = null.
- **전원 CableCategory.specTemplate 확장**(기존 F-CV·FR 등 전원 카테고리에): `번호`·`용량`·`SW상태` 필드 추가. specParams가 값 보관. **다른 도메인 카테고리(UTP 등)엔 없음 → 오염 0.** 새 카테고리 안 만듦.
- `규격(선경)` = 기존 specTemplate, `용도(부하)` = 닿는 자산 도출.
- **새 테이블 0 · 차단기 노드 0 · `Cable.number` 0 · 가짜 카테고리 0.**

### 3.3 방향성 트레이서
```
상류추적(부하 A):
  A 가 닿는 케이블의 상대편으로 이동
  distributor 자산을 만나면 → 그 자산의 입력 케이블(자산쪽 역할=IN) 로 계속
  입력 케이블 없는 distributor(=발전원)에서 종료
```
- `connectionKind`로 분기: distributor=IN 추적 / none=통과 / conduit=후속.
- **`rootKindForCableType`의 AC/DC stop 제거.** **FIBER 는 이번 슬라이스 무변경**(기존 tracePathToRoot/usePortStatus 유지).

## 4. 뷰 — 계통 그리드 (선번장과 동형)

- top-level 뷰에 **"계통"** 추가 (`WorkspacePage.VIEWS`: 현황·평면도·선번장·**계통**).
- 변전소의 distributor 자산을 모아: **분전반(컨테이너)** → **피더 섹션** → **CB(출력 케이블) 행**. 독립 distributor(충전기·UPS)도 자기 출력 행.
```
▸ 분전반 #1
  상용전원반(피더)               사용 5/8
   번호  부하            용량   규격       SW
   1    #1충전기 AC     50A    F-CV 14sq  ON
   2    —              50A    —          OFF
  DC 48V(피더) ...
```
- 컬럼: `번호 · 부하(도출) · 용량 · 규격(category) · SW`. 현황 그리드 통일 양식(h-9·13px·헤더 bg-surface-2).
- **속성 편집은 이 뷰에서**(케이블 속성 폼, 도메인 템플릿 구동) — 연결(그리기/삭제)은 도면. "토폴로지는 도면, 속성은 뷰."

### 4.1 상호작용 (세 그리드 일관 — ★선번장 retrofit 포함)
```
그리드 행 클릭 → 사이드패널 열림(연결탭)  (하이라이트 X)
   연결탭에서 연결 클릭 → 그때 평면도 하이라이트 이동
```
- **선번장 retrofit**: `OfdFiberRegister` 행 onClick 의 `startTrace` 제거(=`setSelectedAssetId`만). 하이라이트는 연결탭(`AssetConnectionsSection`)이 담당.

### 4.2 드로잉 UI
- distributor 자산에 케이블 꽂을 때 **"입력/출력" 지정**(기본 OUT, 입력만 IN) → 케이블 `sourceRole`/`targetRole`. 일반 자산끼린 안 물음.

## 5. 범위 밖 (후속, 명시)

- **conduit + 광/FiberPath·FiberCore 흡수**: OFD=conduit, OPGW=트렁크 케이블, **FiberCore→광 케이블 specParams**(용도/수용내역/융착), **코어번호=`Cable.number`(fiberPortNumber 통합)**, usePortStatus 재작성. *(전원에서 distributor 패턴 증명 후 — FiberCore 방금 출하라 미드스트림 재작성 금지)*
- 선번장·계통 그리드 컴포넌트 완전 일원화(지금 동형 별도 → 제네릭 1개).
- 충전기/UPS/축전지 인벤토리 fieldTemplate(규격·입출력·감시) 정비.
- 접지계통(엑셀에 데이터 없음 — 보류). 전체 SLD 자동작도(분전반→충전기→AC메인 그림).

## 6. 재사용 / 신규

**재사용:** Asset/parentAssetId/Cable/`CableCategory.specTemplate`+`specParams`(케이블 속성 메커니즘 = 이미 존재), `selectionStore`, `AssetConnectionsSection`(연결탭), `usePathHighlightStore`, 워킹카피 단일 커밋, 현황 그리드 양식, `WorkspacePage.VIEWS`.
**신규/수정:** `AssetType.connectionKind`(+시드) / `Cable.sourceRole·targetRole`(+드로잉 UI) / 전원 카테고리 specTemplate 확장(번호·용량·SW) / 방향성 트레이서(distributor, AC·DC rootKind 제거) / 계통 그리드 컴포넌트 + top-level 뷰 / `OfdFiberRegister` 인터랙션 retrofit / `BRANCH` 제거.

## 7. 리뷰/확인 필요

1. **`BRANCH` 제거 범위** — 운영 DB에 분기(BRANCH) 인스턴스가 실제로 있나? 없으면 시드만 제거, 있으면 출력 케이블로 이관 태스크 필요. (구현 시 `SELECT count(*) ... asset_type=BRANCH` 확인)
2. specTemplate에 운영값(SW상태 등 가변 상태)을 두는 것 — 물리 spec(선경)과 한 가방. 가능하나 spec-string 포맷엔 물리 필드만 포함(용량/SW는 표시·편집용 필드).
3. 나머지(CB=케이블 / 속성=카테고리 specTemplate / connectionKind+IN-OUT / FIBER 무변경 / 선번장 retrofit / FiberCore 이관은 후속)는 대화로 확정됨.
