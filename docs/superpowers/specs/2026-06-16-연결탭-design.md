# 연결탭(Connection Tab) Design

**작성일:** 2026-06-16
**상태:** 설계(사용자 승인: 케이블 종류별 그룹 / 특수케이스 없는 단일 함수 / 분전반도 동일) → 사용자 spec 리뷰 → 구현 플랜
**큰 그림:** 상세 패널의 "연결" 탭(현재 빈 플레이스홀더)을 채운다. **클릭한 자산 + 그 안의 보유자산**의 케이블을 **종류별로 출발→도착** 요약하고, 행 클릭 시 도면 하이라이트 + 경로 텍스트, [상세]로 네트워크 토폴로지 모달을 연다. **특수케이스 없는 단일 함수** — 방향(input/output)은 `cableTrace`가 케이블의 역할로 알아서 처리하므로 프론트는 케이스를 나누지 않는다.

---

## 1. 배경 / 현재 구조(정찰 확정)

- **연결 탭은 빈 플레이스홀더:** `src/features/assets/components/AssetInspector.tsx` — `{ label: '연결', render: () => <div ...>연결 UI 재작성 예정</div> }`. `DetailTabs`(`detail/DetailTabs.tsx`)가 탭을 렌더(활성 탭만 lazy).
- **추적(이미 있음):**
  - `src/features/trace/cableTrace.ts` — `cableTrace(startAssetId, cableType, assets, cables): { nodeIds, cableIds, truncated? }`. 같은 `cableType`만, `sourceRole/targetRole`(IN/OUT)·`connectionKind`(distributor/conduit) 역할 기반 순회 → **1대다 자산은 자연히 input쪽만**(피더 본인 출발=양방향, 분기/말단 출발=input만). 링/사이클은 visited 가드.
  - `src/features/trace/traceProjection.ts` — `projectTrace(seedCableId, graph): TraceProjection | null`. `TraceProjection = { seedCableId, nodeIds, cableIds, steps: TraceStep[], nodes, edges, rings, seedFiberEdgeId, truncated }`. `TraceStep = { id, label, isEndpoint, isFiberEdge }`(conduit 슬롯→OFD 접기 포함). 토폴로지(nodes/edges/rings)와 경로 텍스트(steps)를 한 번에 산출.
  - `src/features/trace/traceGraph.ts` — `useTraceGraph(): { graph, isLoading }`. `graph.cables: TraceCable[]`(전역+staged 오버레이), `graph.nameById`.
- **하이라이트/토폴로지(이미 있음):**
  - `src/features/pathTrace/stores/pathHighlightStore.ts` — `usePathHighlightStore`: `startTrace(cableId)`(graph 빌드+projectTrace+하이라이트셋 세팅), `openTopology()`/`closeTopology()`(modalOpen 토글, projection 재사용), `clearHighlight()`. 상태: `active`, `projection`, `tracingCableId`, `highlightedPlacedIds`(노드→배치조상), `highlightedEdgeIds`, `modalOpen`.
  - `src/features/pathTrace/components/PathTraceDetail.tsx` — `projection.steps`를 `node → node ┄ node`(OPGW=점선) 경로 텍스트로 렌더 + "상세" 버튼(`openTopology`). 비활성이면 null.
  - `src/features/network/NetworkTopologyModal.tsx` — `modalOpen && projection`이면 토폴로지(nodes/edges/rings) 렌더. `openTopology()`로 열림.
  - **도면 하이라이트:** `editor/hooks/useCanvas.ts`(설비 dim/bright by `highlightedPlacedIds`) + `connections/components/ConnectionOverlay.tsx`(케이블 glow by `highlightedEdgeIds`). **둘 다 현재 층 설비만 렌더** → 타 층/변전소 노드는 자동으로 하이라이트 안 됨(req6 자동 충족).
- **보유자산 열거(이미 있음):**
  - `src/features/connections/endpointName.ts` — `buildSelfSideChecker(assets, assetId): (id) => boolean`(자기 + 자식 모듈/피더(FEEDER)/conduit 슬롯), `buildEndpointNameResolver(assets): (assetId) => string`(모듈/피더/일반 통합 이름).
- **참고 구현:** `connections/components/AssetConnectionsSection.tsx`(종류별 색점 + source→dest + 클릭 startTrace + 토폴로지 버튼) — 패턴 차용.

## 2. 목표 / 비목표

**목표**
- 연결 탭 = `클릭한 자산 + 보유자산`의 케이블을 **종류별 그룹**으로 **출발→도착** 요약.
- 행 클릭 → 도면 하이라이트 + 하단 경로 텍스트.
- 행 [상세] → 네트워크 토폴로지 모달.
- **링 중복제거(req4):** 같은 경로(노드집합)로 끝나는 케이블은 1행.
- **특수케이스 없는 단일 함수**(분전반/피더/OFD/랙/리프 동일 경로) — 방향은 `cableTrace`가 처리.

**비목표(YAGNI)**
- 분전반/피더 전용 분기(startCircuitTrace 등) — 불필요(케이블별 행 + `cableTrace` 방향 자동).
- 같은 층 필터링 로직 신설 — 캔버스가 현재 층만 렌더하므로 불필요.
- 케이블 생성/편집 — 연결 탭은 읽기 전용 요약(편집은 평면도/계통뷰/선번장).
- 백엔드 변경.

## 3. 데이터 — `useAssetConnections(assetId)`

신규 훅. 단일 경로, 분기 없음.
```ts
export interface ConnectionRow {
  cableId: string;            // startTrace 시드
  fromName: string;           // self 측 끝단 이름(보유자산/자기)
  toName: string;             // 추적 끝단(반대 터미널) 이름
  truncated: boolean;         // 추적이 깊이 제한에 걸렸는지
}
export interface ConnectionGroup {
  key: string;                // cableType(또는 displayGroup) 키
  label: string;              // 전원/네트워크/광/접지/제어 …
  color: string | null;       // 그룹 대표색(케이블 displayColor)
  rows: ConnectionRow[];
}
export function useAssetConnections(assetId: string): ConnectionGroup[]
```
파생(순수, `useTraceGraph` + effective assets 기반):
1. `isSelfSide = buildSelfSideChecker(assets, assetId)`.
2. 후보 케이블 = `graph.cables.filter(c => isSelfSide(c.sourceAssetId) || isSelfSide(c.targetAssetId))`.
3. 각 케이블 → `projectTrace(c.id, graph)`:
   - self 측 = `isSelfSide`를 만족하는 끝 → `fromName`(buildEndpointNameResolver).
   - `toName` = projection의 반대 터미널(steps의 마지막/ self가 아닌 endpoint) 이름.
   - `truncated` = projection.truncated.
4. **링 중복제거:** projection `nodeIds`(정렬)로 시그니처를 만들어 self 그룹 내 동일 시그니처 케이블은 1행(여러 경로가 같은 노드집합으로 닫히면 = 같은 연결).
5. **종류별 그룹화:** 케이블의 종류(전원/네트워크/광 …)로 묶는다. 그룹 키·라벨·색은 케이블 카테고리(displayGroup/displayColor; `useCableCategories`로 categoryId→displayGroup 해소, 없으면 cableType 폴백)에서. 그룹·행 정렬은 라벨/이름 asc.
- 빈 연결이면 빈 배열.

## 4. UI — `AssetConnectionsTab`

`src/features/connections/components/AssetConnectionsTab.tsx`(신규). CONVENTIONS 준수(공용 토큰·타이포, `DetailCard`/`SectionItem` 톤).
- 그룹 헤더: 색점 + 라벨(전원/네트워크/광 …). 종류 없으면 `SectionEmpty`("연결 없음").
- 각 행: `출발 → 도착`(text-sm, truncate) + 우측 `[상세]` 아이콘(또는 작은 버튼). truncated면 끝에 `…` 표시.
- **행 클릭** → `usePathHighlightStore.getState().startTrace(row.cableId)` → 도면 하이라이트(같은 층 자동) + 선택 행 강조(tracingCableId === row.cableId).
- **[상세] 클릭** → `startTrace(row.cableId)` 후 `openTopology()` → `NetworkTopologyModal`.
- 탭 하단에 `<PathTraceDetail />` 1개 — 마지막 클릭한 경로의 step 텍스트(이미 store 구독). 즉 "경로 텍스트"(req3)는 여기 단일 영역에 표시.
- 탭 떠날 때(unmount) `clearHighlight()`(하이라이트 잔상 정리).

## 5. 배선

- `AssetInspector.tsx` 연결 탭 `render` 를 `() => <AssetConnectionsTab assetId={equipmentId} />` 로 교체(플레이스홀더 제거). 탭 `count` = 그룹 내 총 행 수(선택).
- `NetworkTopologyModal` 은 이미 전역(modalOpen 구독)으로 떠 있는지 확인 — 안 떠 있으면 상세 패널/워크스페이스 루트에서 1회 렌더 보장.

## 6. 방향/케이스 일관성 (req2·5)

- `cableTrace`/`projectTrace`가 케이블의 `sourceRole/targetRole` + `connectionKind`로 방향을 결정. 프론트는 **케이블별 행 + `startTrace(cableId)`만** 호출 → 피더 본인 출발은 양방향, 분기/말단 출발은 input만 자연히 처리. **프론트에 분전반 특수분기 없음.**
- 만약 특정 케이스 방향이 틀리면 **`cableTrace` 한 곳만** 수정(프론트 불변).

## 7. 엣지/에러 처리

- **연결 0개:** 빈 상태 안내.
- **추적 truncated(깊이 제한):** 행에 `…`/툴팁로 표시, 경로 텍스트는 잘린 채라도 노출.
- **타 층/타 변전소 끝단:** 경로 텍스트엔 포함, 도면 하이라이트는 캔버스가 현재 층만 → 자동 제외(req6).
- **링/사이클:** projectTrace의 ring + nodeIds 중복제거로 1행.
- **staged(미저장) 케이블:** `graph.cables` 가 오버레이 포함 → 그대로 반영.
- **탭 전환/패널 닫힘:** `clearHighlight()`.

## 8. 테스트 전략

- **`useAssetConnections`(단위):** self set 케이블 수집, projectTrace 끝단 해소(from/to), 종류별 그룹, **링 중복제거**(동일 nodeIds 2케이블 → 1행), 빈 결과. 랙(모듈)·분전반(피더)·OFD(슬롯)·리프 각 케이스가 **같은 함수로** 올바른 행을 내는지.
- **`AssetConnectionsTab`(컴포넌트):** 그룹/행 렌더, 행 클릭 → startTrace(cableId) 호출, [상세] → startTrace + openTopology, 빈 상태, PathTraceDetail 노출.
- **배선(통합):** 연결 탭이 플레이스홀더 대신 AssetConnectionsTab 렌더.
- **빌드/타입:** tsc + 전체 vitest green.
- **수동 스모크:** #1 랙 연결탭 → 모듈별 종류별 출발→도착, 클릭→도면 하이라이트+경로, 상세→토폴로지. 분전반 연결탭이 피더 연결을 동일하게.
