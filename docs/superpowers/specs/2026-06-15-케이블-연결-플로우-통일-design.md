# 케이블 연결 플로우 전면 통일 Design

**작성일:** 2026-06-15
**상태:** 설계(사용자 승인: Q1 우측패널 피커 / Q2 피더 단일 IN 슬롯 / Q3 종류-우선 모달 / Q4 수동 경로 유지 / 단계적 구현) → 사용자 spec 리뷰 → 구현 플랜
**큰 그림:** 흩어진 케이블 생성 진입점·피커·역할선택을 **단일 플로우**(종류→출발→도착)와 **단일 진입 함수**(`startCableConnection`)로 통일하고, 출발/도착 연결점 선택은 **기존 우측 상세 패널 뷰를 그대로 재사용**한다(별도 피커 3종 + 모달 역할선택부 삭제). 피더에 단일 **IN 슬롯**을 추가해 분기(OUT)와 함께 연결점으로 노출한다.

---

## 1. 배경 / 현재 구조(정찰 확정)

현재 케이블 생성은 **출발→도착 순서**이고, 도착을 확정하면 마지막에 `CableSpecModal`이 떠서 **케이블 종류 + IN/OUT 역할**을 한 모달에서 함께 고른다. 도착 자산 종류에 따라 역할선택 UI가 조건부로 끼어들어 **일반화가 안 된다.**

핵심 파일/심볼:
- FSM: `src/features/editor/stores/interactionStore.ts` — phase `selectingSource | pickingSourceModule | drawingPath | pickingTargetModule | selectingSpec`. 액션 `cableActivate()`, `cableSetPendingSource(container,pos)`, `cableSetSource(container,pos,extras?)`, `cableSetPendingTarget`, `cableSetTarget`, `cableAddWaypoint(x,y)`, `cancel()`. extras = `{innerAssetId?, slotId?, coreNumber?}`.
- 도구/모드 동기화: `src/features/editor/stores/editorStore.ts` — `setTool('cable')`, `cancelCableDrawing()`(단일 종료 choke-point), `preselectedCableDisplayGroup`/`setPreselectedCableDisplayGroup`. `FloorPlanEditor.tsx`(~line 140) effect 가 `tool==='cable'`↔`cableActivate/cancel` 동기화.
- 종류+역할 모달: `src/features/editor/components/modals/CableSpecModal.tsx` — 카테고리 그리드(`useCableCategories`, `preselectedGroup` 필터) + **distributor 일 때만** IN/OUT 역할선택부(~line 213–233). `isDistributor/isConduit`, 엔드포인트 해소(slot>inner>container).
- 피커 디스패처: `src/features/editor/components/CableEndpointPickerHost.tsx` → `RackModulePicker`(랙) / `CircuitPicker`(피더) / `SlotCorePicker`(OFD+코어). 각 onSelect 가 `cableSetSource/Target(extras)` 호출.
- 캔버스 클릭: `src/features/editor/hooks/useCanvasEvents.ts`(~319–346) — selectingSource/drawingPath 에서 자산 클릭 → `needsEndpointPicker(kindOf(eq))` 면 `cableSetPending*`, 아니면 `cableSet*` 직접.
- 키보드/사이드바 진입점: `useEditorKeyboard.ts`('3'/'C'/ESC), `EditorInsertBar.tsx`(~line 111, 그룹 pill → `setTool('cable')`+`setPreselectedCableDisplayGroup`).
- 분산 진입점: `FeederCircuitsPanel.tsx`(~line 51) "분기 추가" → `setTool('cable')`+`gotoAsset`(출발 자동주입 없음, 캔버스 재클릭 필요).
- 데이터 모델: `editorStore.LocalCable` — `sourceAssetId/targetAssetId`(다형, 우선순위 slot>inner>container), `sourceRole/targetRole: 'IN'|'OUT'|null`, `number`(코어/CB번호), `categoryId/specParams`. 저장 `substationStore.stageCableCreate`.
- 연결점 뷰(이미 존재, 재사용 대상): `DistributionCircuits`(분전반→피더 카드), `FeederCircuitsPanel`+`BreakerRail`(피더 분기 6×4, CB=OUT, number=CB), `SlotPortsPanel`+`PortGrid`(경로슬롯 포트), `SlotRailGrid`(랙/OFD 레일). 파생: `buildFeederCircuits`/`feederGridSlots`, `buildSlotPorts`.
- 매핑: `kindOf(asset)`(placementKind→EquipmentKind), `needsEndpointPicker`(RACK/OFD/DISTRIBUTION), `AssetType.connectionKind`(distributor/conduit/null), `floorAnchor`/`floorTargetFor`.

**데이터모델 사실:** distributor(피더)는 `IN/OUT` 역할을 **이미 지원**한다. 지금은 OUT(분기 CB)만 쓰일 뿐 — "피더 Input"은 새 개념이 아니라 **role=IN 케이블 + UI**만 추가하면 된다.

## 2. 목표 / 비목표

**목표**
- 케이블 생성 순서를 **종류 → 출발 → 도착**으로 재배치.
- 출발/도착 연결점 선택을 **우측 상세 패널 뷰 재사용**(pick 모드)으로 통일 → 피커 3종 + `CableEndpointPickerHost` + 모달 역할선택부 삭제.
- 모든 진입점을 **단일 함수 `startCableConnection(opts)`** 로 수렴, 진입점이 출발을 알면 **자동주입**(종류→도착만).
- 피더에 **단일 IN 슬롯** 추가(데이터=role=IN 케이블), 피더 패널·피커 모두에 노출.
- 수동 경로(waypoint) 그리기 유지.

**비목표(YAGNI)**
- 다중 IN(이중공급) — 데이터모델은 허용하나 UI 는 단일 IN 슬롯만.
- 경로 자동 라우팅 알고리즘(직선 + 기존 waypoint 편집 유지).
- 경로슬롯/랙의 IN/OUT 역할 신설(현행 유지: 경로슬롯 OUT 고정, 랙모듈 무역할).
- 백엔드 스키마 변경(기존 role/number/specParams 그대로).

## 3. 데이터 모델 — `EndpointRef`

모든 엔드포인트를 단일 타입으로 표현(기존 extras + role 확장). FSM·피커·진입점이 공유.
```ts
export interface EndpointRef {
  containerAssetId: string;        // 평면도에 배치된 anchor 설비(floorAnchor 기준)
  position: { x: number; y: number }; // 경로/프리뷰용 컨테이너 중심
  innerAssetId?: string | null;    // 하위 자산(랙모듈 / 피더 FEEDER id)
  slotId?: string | null;          // 경로슬롯(conduit) id
  coreNumber?: number | null;      // 광 코어 번호
  role?: 'IN' | 'OUT' | null;      // distributor 연결점 역할(피더 IN 슬롯 / 분기 OUT)
  number?: number | null;          // 분기(CB) 번호 등
}
```
- 엔드포인트 assetId 해소(기존 규칙 유지): `slotId ?? innerAssetId ?? containerAssetId`.
- 피더 IN 슬롯 = `{containerAssetId: DIST anchor, innerAssetId: feederId, role:'IN'}`.
- 피더 분기 = `{innerAssetId: feederId, role:'OUT', number: CB번호}`.
- 경로슬롯 포트 = `{slotId, coreNumber, role:'OUT'}`(현행).
- 평면 설비 = `{containerAssetId}`(role/inner 없음).

## 4. 새 FSM 플로우

`interactionStore` cableDrawing phase 재정의(`drawingPath` 중 도착 자산 클릭이 도착 선택을 겸함 — 별도 selectingTarget phase 없음):
```
selectingType → selectingSource → pickingSourceEndpoint → drawingPath ──(도착 클릭)──→ pickingTargetEndpoint → (생성)
                                   (하위 없으면 생략)                      (하위 없으면 생략, 즉시 생성)
```
- **cableActivate(opts?: { source?: EndpointRef; cableType?: CableType; group?: CableDisplayGroup })**
  - `cableType` 있으면 종류 모달 생략, 없으면 `phase='selectingType'`(CableSpecModal 종류 전용).
  - 종류 확정 후: `source` 주입돼 있으면 source 세팅 후 `phase='drawingPath'`(도착만 남음), 아니면 `phase='selectingSource'`.
- **selectingSource / selectingTarget**: 캔버스 자산 클릭 → `needsEndpointPicker(kindOf)` 면 `phase='picking*Endpoint'`(+ 우측 패널 그 자산으로 open), 아니면 `EndpointRef` 즉시 확정.
- **pickingSourceEndpoint / pickingTargetEndpoint**: 우측 패널(아래 §5)에서 연결점 onPick → `EndpointRef` 확정. source→`drawingPath`, target→**즉시 케이블 생성**(종류 이미 정해짐).
- **drawingPath**: `cableAddWaypoint`(캔버스 빈곳 클릭), 백스페이스 undo, 다른 자산 클릭 → 도착 분기.
- **cancel()**: ESC/패널 닫기/모달 취소 모두 → idle (`editorStore.cancelCableDrawing` 단일 종료 유지).

마지막 `selectingSpec` 모달 제거(종류는 맨 앞). 역할은 연결점 선택에 내재.

## 5. 우측 상세 패널 = 연결점 피커 (pick 모드)

`pickingSourceEndpoint|pickingTargetEndpoint` 동안, 평면도 우측 패널이 클릭한 자산의 **기존 뷰**를 그대로 렌더하되 "연결 모드"로 동작.

- **pick 모드 신호:** `interactionStore` 의 phase 에서 파생한 `useCablePick()` 훅 → `{ active: boolean; onPick(ref: EndpointRef): void }`. 활성 시 연결점 클릭이 편집/선택 대신 `onPick` 호출.
- **재귀(컨테이너→하위→연결점)는 기존 패널 드릴다운 그대로:**
  - 분전반(DIST): `DistributionCircuits` — 피더 카드 클릭 → 그 피더 패널.
  - 피더: `FeederCircuitsPanel` — **IN 슬롯 / 6×4 분기** 중 클릭 → `onPick({innerAssetId:feederId, role/number})`. (분기는 빈칸 클릭=새 CB번호 자동, 점유칸=그 CB.)
  - 경로슬롯/OFD: `SlotRailGrid`(슬롯 선택) → `SlotPortsPanel`/`PortGrid`(포트 클릭) → `onPick({slotId, coreNumber, role:'OUT'})`.
  - 랙: `SlotRailGrid`+`ModuleCell` — 모듈 클릭 → `onPick({innerAssetId:moduleId})`.
  - 하위/연결점 없는 평면 설비: 패널 없이 캔버스 클릭에서 즉시 확정(§4).
- 각 뷰는 **선택적 `onPick` prop**(또는 `useCablePick` 소비)을 받아, 없으면 기존(편집) 동작. 한 컴포넌트가 보기·편집·피커를 겸한다.
- **삭제:** `RackModulePicker`, `CircuitPicker`, `SlotCorePicker`, `CableEndpointPickerHost`, `CableSpecModal` 의 역할선택부(`RoleSelector` 등) 및 `computeTargetRole`/`isDistributor`/`isConduit` 의 역할 분기.

## 6. 피더 IN 슬롯

- **데이터:** 피더(FEEDER, distributor)에 닿는 `role==='IN'` 케이블 = 그 피더의 공급(Input). 단일(개념상 1개; 2개 이상이면 최신/첫 1개 표시 + 경고는 비목표).
- **파생:** `feederCircuits.ts` 에 `buildFeederInput(feeder, cables, nameById): { cableId, sourceName } | null` 추가(roleAt==='IN' 케이블 1개).
- **UI(`FeederCircuitsPanel`):** 6×4 분기 그리드 **위에 단일 IN 슬롯**(상단). 점유 시 공급원 이름 + 삭제, 빈 시 "＋ 입력 연결"(=`startCableConnection({source:{feeder, role:'IN'}})`). 디자인 언어는 기존 흰 모듈/슬롯 톤(CONVENTIONS).
- **피커일 때:** IN 슬롯과 분기 그리드가 둘 다 onPick 대상 — IN 에 꽂으면 role=IN, 분기에 꽂으면 role=OUT+CB번호.

## 7. 진입점 통일 — `startCableConnection`

```ts
// src/features/editor/cableConnection.ts (신규, React 밖에서도 호출 가능)
export function startCableConnection(opts?: {
  source?: EndpointRef;            // 주면 출발 자동주입 → 종류→도착만
  cableType?: CableType;           // 주면 종류 모달 생략
  group?: CableDisplayGroup;       // 종류 모달 사전 필터
}): void
// 내부: editorStore.setTool('cable') + (group? setPreselectedCableDisplayGroup) + interaction.cableActivate(opts)
//       source 의 floorAnchor 로 평면도 진입(gotoAsset) 필요 시 수반.
```
- **모든 진입점이 이 함수로 수렴**(직접 `setTool('cable')`/`cableActivate` 호출 제거):
  - 키보드 '3'/'C', 사이드바 그룹 pill(→ `{group}`), 피더 "분기 추가"(→ `{source:{feeder, role:'OUT'}}`), 피더 IN "입력 연결"(→ `{source:{feeder, role:'IN'}}`), 향후 모든 "여기에 연결" 버튼.
- **자동주입 효과:** 출발이 정해진 진입점은 종류→도착만. "분기 추가"는 도착(부하)만 고르면 끝.

## 8. 삭제/정리 대상(통일의 이득)

- 컴포넌트: `RackModulePicker.tsx`, `CircuitPicker.tsx`, `SlotCorePicker.tsx`, `CableEndpointPickerHost.tsx`.
- `CableSpecModal`: 역할선택부·distributor/conduit 분기 제거(순수 종류 선택으로 축소, 위치만 맨 앞).
- 흩어진 `setTool('cable')`/`cableActivate` 직접 호출 → `startCableConnection` 1곳.
- `interactionStore` extras 의 임시 분기 정리(EndpointRef 로 단일화).

## 9. 단계 분해(플랜 예고, 각 단계 후 검증·머지)

1. **FSM 종류-우선 재배치:** `selectingType` 를 맨 앞으로, `CableSpecModal` 종류 전용화(역할선택부 제거), 도착 확정=즉시 생성. 수동 경로 유지. `EndpointRef` 도입(내부 치환). 기존 피커는 잠정 유지.
2. **우측 패널 pick 모드:** `useCablePick` + 각 뷰(`DistributionCircuits`/`FeederCircuitsPanel`/`BreakerRail`/`PortGrid`/`SlotRailGrid`)에 onPick. `useCanvasEvents` 가 컨테이너 클릭 시 패널 open+pick 모드. 피커 3종 + `CableEndpointPickerHost` **삭제**.
3. **피더 IN 슬롯:** `buildFeederInput` + `FeederCircuitsPanel` 상단 IN 슬롯(보기/삭제/입력연결) + 피커에서 IN/분기 선택.
4. **진입점 통일:** `startCableConnection` 신설 + 전 진입점 이관(자동주입) + 흩어진 호출/임시코드 삭제.

## 10. 엣지/에러 처리

- **출발=도착 동일 자산:** 거부(자기연결 방지) — 기존 가드 유지.
- **종류 모달 ESC / 패널 닫기 / 캔버스 ESC:** 모두 `cancelCableDrawing`(단일 종료) → idle+tool=select(먹통 방지).
- **자동주입 출발이 타 층/타 변전소:** `gotoAsset` 로 평면도 진입 후 종류→도착.
- **피더 IN 이미 존재 시 또 연결:** 신규 IN 케이블 추가는 허용하되 UI 는 1개만 표시(다중 IN 비목표).
- **빈 분기 클릭(피커):** 다음 빈 CB번호 자동 배정(기존 feederGridSlots 위치 규칙).
- **connectionKind 없는 평면 설비:** 패널 없이 즉시 엔드포인트(역할 null).

## 11. 테스트 전략

- **FSM(단위):** cableActivate(opts) 분기(type 유무·source 유무) → phase 전이, 도착 확정 시 stageCableCreate 인자(종류/역할/번호). selectingType→source→drawingPath→target 순서.
- **EndpointRef 해소(단위):** slot>inner>container 우선순위, 피더 IN/분기/포트/평면 각 케이스 → LocalCable 필드.
- **pick 모드 뷰(컴포넌트):** onPick 주면 연결점 클릭이 onPick(ref) 호출·편집 미발생; 없으면 기존 동작. 피더 IN/분기, 포트, 랙모듈 각각.
- **buildFeederInput(단위):** roleAt IN 케이블 해소, 없음=null.
- **진입점(통합):** startCableConnection({source}) → 종류→도착만; 그룹 pill → 필터; 분기/입력 버튼 → 역할 주입.
- **삭제 회귀:** 피커 3종/PickerHost 참조 0, 역할선택부 제거 후 빌드·전체 vitest green, tsc clean.
- **수동 스모크:** 평면도에서 랙→피더분기, 피더IN←타설비, OFD포트↔포트, "분기 추가"=도착만.
