# 자산 role 단일화 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `AssetType` 분류를 `role` 단일축으로 통일하고 레거시 `placementKind`·`connectionKind`·`group` 컬럼을 제거한다.

**Architecture:** `role`은 기존 축의 결정적 superset(이미 시드가 채움)이라 백필 불필요. C5와 동일한 2단계: 모든 reader를 role로 이전(소비처 0) → 컬럼 드롭. 평면도 배치 contract의 `kind`/`PlacementKind`는 폐기하고 `role` 직접 사용.

**Tech Stack:** Prisma/PostgreSQL, Express, Zod, React 18 + TS, Zustand, vitest.

## Global Constraints

- **개발 DB:** `docker compose -f docker-compose.dev.yml up -d`. docker build / 기본 compose / 프로덕션 빌드 금지. (CLAUDE.md)
- **마이그레이션:** dev DB 에 `cd backend && npx prisma migrate dev` (TTY 데이터손실 프롬프트가 막으면 손수 `migration.sql` 작성 후 `migrate deploy` + `prisma generate`, `migrate status`로 확인).
- **브랜치:** main 에서 분기한 `feat/asset-role-unify`. **선행:** main 워킹트리에 미커밋된 랙 콤보박스 변경 2파일(`CategoryComboboxPopover.tsx`, `useRackModuleCategories.ts`)이 있다 — 이 브랜치 분기 전에 별도 커밋하거나 이 브랜치 첫 커밋으로 포함한다(작업 시작 시 정리).
- **회귀 게이트:** 백엔드 변경 → `cd backend && npx tsc --noEmit && npx vitest run`. 프론트 변경 → `cd frontend && npx tsc --noEmit && npx vitest run`(현재 513 기준).
- **role 매핑(불변, 시드 deriveRole 그대로):** RACK→`rack`, OFD→`ofd`, DIST→`panel`, GROUNDING/HVAC→`standalone`, conduit→`slot`, distributor→`feeder`, 그외→`device`.
- **top-level 설비 판별:** 이미 쿼리의 `parentAssetId: null`로 충분 — `placementKind not null` 필터는 그것과 중복이라 **삭제**한다(role 서브셋 상수 신설 금지). 꼭 역할 비교가 필요하면 리터럴(`role === 'rack'`)로 인라인.
- **모듈 카테고리:** = `role === 'device'` (기존 `placementKind: null`의 OFD-SLOT 오포함 교정).
- **드롭 대상:** `AssetType.placementKind`·`connectionKind`·`group`(텍스트). `role`·`code`·`categoryId`·`displayColor`·`fieldTemplate` 등은 유지.
- **비목표:** role enum 값 불변(7종), AssetCategory 구조 불변, 케이블 무관, role 사용자 편집화 없음.
