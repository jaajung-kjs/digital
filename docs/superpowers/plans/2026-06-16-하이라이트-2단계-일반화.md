# 하이라이트 2단계 일반화 (선택강조 vs 경로집중) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 하이라이트를 "선택강조(soft)"와 "경로집중(trace)" 2단계로 일반화하고, 카메라 이동과 토폴로지 모달을 하이라이트에서 분리해 토폴로지 종료 시 평면도가 딤·이동된 채 남는 버그를 없앤다.

**Architecture:** `pathHighlightStore`에 `mode: 'select' | 'trace' | null` 을 추가한다. 단순 선택(`useSelectionHighlight` → `highlightDiagram`)은 `select` 모드(글로우만, 딤·카메라 없음), 명시적 추적(`startTrace`/`startCircuitTrace`)은 `trace` 모드(강한 0.2 딤 + 카메라 정렬)로 매핑한다. 토폴로지 모달은 `projection`+`modalOpen`만 쓰므로(`active`/`highlightedEdgeIds` 안 읽음), 모달 데이터 계산(`prepareTopology`)을 평면도 하이라이트/카메라와 완전히 분리한다 — 모달을 열고 닫아도 평면도 상태는 불변.

**Tech Stack:** React + Zustand + canvas 2D 렌더, Vite, Vitest, TypeScript. 개발 서버: `docker compose -f docker-compose.dev.yml up -d` + `npm run dev`.

**검증 전략(중요):** 이 변경의 핵심은 canvas 렌더(딤)와 카메라 이동·모달 상호작용으로, 단위 테스트로 잡기 어렵고 store가 무거운 비동기 의존(queryClient/api/workingCopy)을 가진다. 따라서 1차 검증은 **(a) `npm run build` 타입체크 + (b) 기존 테스트 그린 유지 + (c) 브라우저 수동 검증(두 증상 재현→해소 확인)** 으로 한다. 가짜 단위 테스트는 만들지 않는다. 순수 로직(`resolveHighlight`)은 이미 회귀 테스트가 있으면 유지한다.

```ts
async function loadProjection(
  cableId: string,
): Promise<{ ok: true; projection: TraceProjection; ids: Set<string> } | { ok: false; error: string }> {
  const slimAssets = await fetchAllSlimAssetsCached(queryClient);
  const globalCables = await queryClient.fetchQuery({
    queryKey: ['cables'], staleTime: 30_000,
    queryFn: async () => (await api.get<{ data: TraceCableInput[] }>('/cables')).data.data,
  });
  const wc = useSubstationWorkingCopy.getState();
  const graph = buildTraceGraph({
    slimAssets: slimAssets as SlimAssetDTO[],
    globalCables,
    stagedAssets: wc.effectiveAssets() as never[],
    stagedCables: wc.effectiveCables() as unknown as TraceCableInput[],
    deletes: [...wc.overlays.cables.deletes, ...wc.overlays.assets.deletes],
  });
  const projection = projectTrace(cableId, graph);
  if (!projection) {
    return { ok: false, error: '시드 케이블을 찾을 수 없습니다. 삭제되었거나 캐시가 갱신되지 않았을 수 있습니다.' };
  }
  return { ok: true, projection, ids: new Set(projection.nodeIds) };
}
```

2. 연결탭에서 컴포넌트의 토폴로지(↗) 버튼 클릭 → 모달 열림.
3. 모달 X 로 닫기.
4. **기대:** 평면도 줌/팬이 2번 직전과 동일(카메라가 경로로 안 튐).

- [ ] **Step 5: 증상 #2 — 종료 후 설비 투명/블러 없음 확인**

2. **기대:** 경로 외 설비가 0.2 투명으로 깔리지 않음(선택 강조는 글로우/케이블 강조로만 표현). 모든 설비 정상 불투명.

- [ ] **Step 6: 회귀 — 명시적 경로 추적은 여전히 딤+카메라 동작 확인**

1. 연결도(설비 상세) 또는 캔버스 우클릭/단축키로 케이블 **추적**(startTrace) 실행.
