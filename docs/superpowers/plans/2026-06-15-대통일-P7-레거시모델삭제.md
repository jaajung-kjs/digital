# 연결 모델 대통일 — P7 레거시 광 모델 삭제 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** 신모델(슬롯/OPGW/OUT)이 광의 유일 진실이 된 상태(P3a/P3b/P6 + 커오버 완료, dev fiber_paths=0·fiber_cores=0·fiberPathId 케이블 0)에서, 죽은 레거시 `FiberPath`/`FiberCore` 모델·`Cable.fiberPathId`/`fiberPortNumber` 컬럼·`/fiber-paths` 라우트·관련 서비스/스키마/타입을 **전부 삭제**한다. 운영 실데이터 없음 → 2단계 마이그레이션 불필요(클린 삭제).

**Architecture:** 프론트 죽은 참조 제거(working-copy fiberPaths/fiberCores 컬렉션·스테이지·effective·커밋 델타, 광 케이블 타입의 fiberPathId/fiberPortNumber/fiberPathLabel 필드, fiber/types·label 죽은분) → 백 서비스/라우트/스키마/커밋/워킹카피 fiber 처리 + 케이블 DTO fiber 필드 + assertOfdFiberPath/buildFiberPathLabel 제거 → Prisma 모델·컬럼·관계 drop 마이그레이션. **유지:** `TraceEdge.fiberPathId/fiberPortNumber`(내부 그래프 타입, DB 컬럼 아님), `useOfdDirectory`(광 전용 아님 — 단 사용처 0이면 제거).

**Tech Stack:** Node/Prisma/TS, React/TS, Vitest.

**참조:** P7 정찰 맵(이 플랜의 타깃 출처). 신모델: 슬롯=conduit 자식, OPGW=IN-IN, OUT=number 코어.
