# C2 — 프론트 케이블 트레이스 cableType 제거 (그룹/구조 기반) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** 프론트엔드 트레이스/렌더/그룹핑이 `cableType`(FIBER 등)을 **전혀 읽지 않게** 만든다. "같은 종류 추적"은 **사용자 그룹**(categoryId→group)으로, "OPGW 판별"은 **연결 구조(슬롯↔슬롯 IN-IN)**로, 색은 **그룹 색**으로 대체.

**Architecture:** C1에서 `CableCategory` DTO가 이미 `groupId`/`groupColor`/`groupName`을 제공한다(`useCableCategories`). 프론트는 케이블의 `categoryId`로 그룹을 해소해 트레이스 필터·색·그룹핑에 쓴다. `cableType` 컬럼/enum 자체 드롭은 C5(여기선 **읽기만 제거**, 생성 시점은 컬럼 NOT NULL 이라 C5까지 계속 'FIBER' 등 기록).

**Tech Stack:** React 18 + TS, React Query(useCableCategories), vitest. 백엔드 변경 없음.

- [ ] **Step 1: useAssetConnections `makeCategoryGroupOf`** — categoryId→category(=groupId/groupName/groupColor) 로 그룹 키/색 산출. `CABLE_TYPE_DISPLAY_GROUP[cable.cableType]` 폴백(25-31행) 제거 → categoryId 없으면 `{ key: '기타', color: null }`.
```ts
export function makeCategoryGroupOf(categories: CableCategory[]): (cable: { categoryId?: string | null; color?: string | null }) => CategoryGroup {
  const byId = new Map(categories.map((c) => [c.id, c]));
  return (cable) => {
    const cat = cable.categoryId ? byId.get(cable.categoryId) : undefined;
    if (cat) return { key: cat.groupName ?? '기타', label: cat.groupName ?? '기타', color: cat.groupColor ?? null };
    return { key: '기타', label: '기타', color: null };
  };
}
```
