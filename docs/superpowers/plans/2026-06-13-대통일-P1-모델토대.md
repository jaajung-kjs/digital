# 연결 모델 대통일 — P1 모델 토대 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 대통일의 토대(범용 `Cable.number`, 광슬롯 conduit 자산종류, OPGW 케이블 카테고리)를 *추가만* 해서 이후 단계(범용 트레이스·마이그레이션)가 올라탈 자리를 만든다. 기존 FiberPath/fiberPortNumber 와 **병행**(아무것도 안 지움 = 저위험).

**Architecture:** `Cable.number`(범용 슬롯/포트/CB 번호, fiberPortNumber 일반화) 컬럼 + 커밋/프론트 배선; 광슬롯 AssetType(connectionKind=conduit, OFD 자식); OPGW CableCategory. 전부 nullable/추가라 기존 동작 무변경.

**Tech Stack:** Prisma/PostgreSQL, Node/Zod, React/TS, Vitest.

**참조 스펙:** `docs/superpowers/specs/2026-06-13-연결모델-대통일-design.md` (§2.2, §5)

## 환경 규칙 (공통)
- DB: dev compose만(`docker compose -f docker-compose.dev.yml up -d`, 이미 떠있음). **build/기본 up 금지.**
- 마이그레이션: `cd backend && npx prisma migrate dev --name <name> && npx prisma generate`. seed 가 무관한 `gangwonSubstations.ts`(stale cable.upsert)에서 깨질 수 있음 — 그건 사전 부채, **psql 로 결과 검증**하고 무시.
- 백 테스트 `cd backend && npx vitest run <file>`, 프론트 `cd frontend && npx vitest run <file>`, 빌드 `cd frontend && npm run build`.
- psql: `docker exec ict-twin-postgres psql -U postgres -d ict_digital_twin -c "<sql>"`.
