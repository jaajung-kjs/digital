## S4: preset JSON 데이터 마이그레이션 (code→id, code 살아있을 때)

**Files:** Create `backend/prisma/migrations/<ts>_rackpreset_modules_code_to_id/migration.sql` (or a guarded TS one-off).

**Interfaces:** existing `rack_presets.modules[].categoryCode` → `categoryId` (asset_types.code → id). Must run while `code` column still exists (before S5).

- [ ] **Step 1: 변환 마이그레이션 작성**
For each rack_preset row, rewrite `modules` jsonb: for each element, set `categoryId` = (SELECT id FROM asset_types WHERE code = element->>'categoryCode'), remove `categoryCode`. SQL (Postgres jsonb):
```sql
UPDATE rack_presets rp SET modules = sub.new_modules
FROM (
  SELECT rp2.id,
    jsonb_agg(
      (elem - 'categoryCode') || jsonb_build_object('categoryId', at.id)
    ) AS new_modules
  FROM rack_presets rp2,
       jsonb_array_elements(rp2.modules) elem
       LEFT JOIN asset_types at ON at.code = elem->>'categoryCode'
  GROUP BY rp2.id
) sub
WHERE rp.id = sub.id;
```
(Unmapped codes → categoryId null; log via a SELECT first if desired.)

`cd backend && npx prisma migrate dev --name rackpreset_modules_code_to_id` (or hand-write + deploy). Verify: `SELECT modules FROM rack_presets LIMIT 3` shows categoryId, no categoryCode. `migrate status` clean.
