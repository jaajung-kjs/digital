# C5 — 레거시 케이블 컬럼/enum 드롭 + 데드코드 0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 케이블 대통일의 마지막 단계 — `cableType` enum·`CableCategory.code`·`displayGroup`·하드코딩 색/그룹 맵·`getCableTypeFromMaterial` 등 레거시 잔재를 코드·DTO·DB 스키마에서 완전히 제거한다(데드코드 0).

**Architecture:** 비가역 컬럼 드롭이므로 **2단계**다. **Phase A(소비처 0):** 레거시 컬럼을 읽는/내보내는 모든 코드(백엔드 DTO·스키마, 프론트 직렬화·렌더·트레이스·상수)를 제거해 컬럼을 "아무도 안 읽는" 상태로 만든다 — 컬럼은 DB에 그대로, Prisma 클라이언트도 그대로라 `tsc`·기존 vitest 전수 통과가 게이트. **Phase B(드롭):** Prisma 스키마에서 컬럼/enum을 지우고 마이그레이션을 적용하며, NOT NULL 이라 Phase A 동안 백엔드가 써 넣던 placeholder(`cableType`·자동생성 `code`)를 **드롭과 같은 커밋에서** 제거한다(throwaway nullable 마이그레이션 회피).

**Tech Stack:** Prisma/PostgreSQL, Express, Zod, React 18 + TS, Zustand, React Query, vitest + @testing-library/react.

## Global Constraints

- **개발 DB:** `docker compose -f docker-compose.dev.yml up -d` 로 띄운 dev DB 사용. `docker compose build`/기본 `docker compose up` 금지. (CLAUDE.md)
- **마이그레이션 적용:** dev DB 에 `npx prisma migrate dev` 로 적용·생성. 프로덕션 빌드/이미지 빌드 금지.
- **회귀 게이트:** 각 Phase A 태스크는 `cd frontend && npx tsc --noEmit` (프론트 수정 시) 또는 `cd backend && npx tsc --noEmit` (백엔드 수정 시) + 관련 vitest 통과로 끝낸다. Phase A 종료 체크포인트는 **프론트 전체 vitest(현재 497 통과) + 백엔드 전체 vitest 전수 통과**가 게이트.
- **비가역 드롭 안전:** Phase B 진입 전 체크포인트에서 "소스에 드롭 대상 컬럼 참조 0" 을 grep 으로 증명한다(아래 체크포인트 절차).
- **UI 규칙:** 프론트 UI 변경은 `frontend/src/components/ui/CONVENTIONS.md` 준수 — 공용 컴포넌트(DetailCard·EditableField·Button) 재사용, `text-xs` 미만 금지.
- **placementKind 는 비목표.** `AssetType.placementKind` 는 자산측 컬럼으로 별개 정리 대상 — 이 플랜에서 건드리지 않는다.

**드롭 대상 (Phase B):**
- `CableCategory`: `code`(@unique NOT NULL), `displayGroup`, `displayColor`, `specTemplate`, `unit`, `description`, `iconName`
- `Cable`: `cableType`(enum NOT NULL), `color`, `label`
- `enum CableType { AC DC LAN FIBER GROUND }`

**유지:** `Cable.categoryId`·`specParams`·`sourceRole`/`targetRole`·`number`·`pathPoints`/`pathLength`/`bufferLength`/`totalLength`. `CableCategory.{id,name,groupId,sortOrder,isActive,timestamps}`. `AssetType.code`·`RackModule.categoryCode`(= AssetType.code, 케이블과 무관 — **유지**).

function SpecCell({ cableId, categoryId, name }: { cableId: string; categoryId: string | null; name: string }) {

      ariaLabel="규격"

          categoryName: cat?.name ?? null,
          groupId: cat?.groupId ?? null,
          groupColor: cat?.groupColor ?? null,

        });
