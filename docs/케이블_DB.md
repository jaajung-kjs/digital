# 케이블 DB (Cable Database)

> 138개 KEPCO ICT 공사 설계서 Excel 분석 + 업계 표준 규격 교차 검증
> 분석일: 2026-04-09
>
> **specTemplate 정의**: [스키마_리팩토링_계획.md](스키마_리팩토링_계획.md) §3-1 참조
> **아키텍처**: 파라미터 기반 동적 규격 선택 — 종류별 `specTemplate`으로 선경(SQ)/심수(C)/커넥터 등 파라미터를 UI에서 선택하면 규격 문자열이 자동 생성되고, Material 레코드가 on-demand 생성됨.

## 요약

- **총 16종 케이블** (DB 기준) — 모든 ICT 공사 케이블 커버
- **총 84개 표준 규격** — 1,491종 원본 표기를 정규화한 결과
- **평균 변형률 17:1** — 같은 규격이 평균 17가지 다른 표기로 작성됨
- **분석 대상**: 137개 파일, 2,516건 케이블 레코드

> v3: Excel 분석 코드(기능 그룹)를 DB 코드(시공유형 기반)로 재분류.
> CBL-PWR → CBL-FCV/CBL-FR/CBL-VCT/CBL-HIV 분리, CBL-GND → CBL-IV/CBL-BARE 분리,
> CBL-STP → CBL-UTP에 shield 파라미터 통합, CBL-IO → CBL-SIG에 통합.

## 1. 전원케이블 (`CBL-PWR`) → DB: `CBL-FCV` + `CBL-FR` + `CBL-VCT` + `CBL-HIV`

- **파라미터 구조**: `[종류] [선경]SQ [심수]C`

## MaterialAlias 매핑 규칙

DB 시드 시 각 표준 규격(Material)에 대해 아래 패턴의 변형 표기를 MaterialAlias로 등록:

| 변형 패턴 | 예시 |
|----------|------|
| 구분자 차이 | `6SQ/2C` vs `6SQ*2C` vs `6SQ 2C` vs `6SQ×2C` |
| 대소문자 | `Cat.5E` vs `CAT.5E` vs `cat5e` |
| 공백 유무 | `F-CV6SQ2C` vs `F-CV 6SQ 2C` |
| 단위 표기 | `6SQ` vs `6㎟` vs `6sq` |
| 이름 변형 | `광점퍼코드` vs `점퍼코드` vs `JUMPER CORD` vs `OJC` |
| 순서 변경 | `F-CV, 2C, 6SQ` vs `F-CV 6SQ 2C` |
| 접두사 차이 | `FR-CV` vs `F-CV` vs `FCV` |
