> ICT 디지털 트윈 — 도면 기반 공량·자재 자동 산출
> 작성일: 2026-04-10

---

## 1. 시스템 목표

도면의 Before(준공 상태)/After(설계 상태) 비교를 통해 **노무량(공량)과 자재수량을 자동 산출**한다.

```
[준공 도면] → [사용자가 설계 수정] → [Before/After Diff] → [공량·자재 자동 산출]
```

### 설계 원칙

1. **기존 기능 100% 유지** — 설비 사진, 점검이력, 경로추적, 토폴로지, 3D뷰 등 모든 기존 기능 유지
2. **하나의 데이터** — 공사설계자와 일반사용자가 같은 데이터를 본다. 이중 관리 없음
3. **자재 선택이 카테고리 선택을 대체** — 추가 단계 최소화 (1~2단계), 최근사용/프리셋으로 1클릭
4. **점진적 확장** — 기존 테이블 삭제 없이, 새 컬럼/테이블 추가 방식

## 3. DB 스키마 변경

### 3-1. 변경 전략: 기존 유지 + 확장

기존 테이블의 관계를 유지하면서 새 컬럼/테이블을 추가한다.

**기존 관계 유지 (변경 없음):**
- Equipment → EquipmentPhoto (설비 사진)
- Equipment → MaintenanceLog (점검 이력)
- Equipment → Port (포트 관리)
- Equipment → Cable (sourceCables, targetCables — 케이블 연결/시각화)
- Equipment → FiberPath (광경로)
- Cable → pathPoints (배선도)
- CableTrace Service (경로추적, 링감지, 토폴로지)
- AuditLog (변경 이력)
- 조직 계층 (Headquarters → Branch → Substation → Floor → Room)

### 4-2. specTemplate 아키텍처

파라미터 기반 동적 규격 선택. 종류별 `specTemplate`으로 UI에서 파라미터를 선택/입력하면 규격 문자열이 자동 생성되고, Material 레코드가 on-demand 생성된다.

```typescript
interface SpecParam {
  key: string;           // "sq", "cores", "cat"
  label: string;         // "선경(SQ)", "심수(C)"
  inputType: "select" | "number" | "text";
  options?: (string | number)[];  // select일 때 선택지
  unit?: string;         // "SQ", "C", "mm"
  required?: boolean;    // default: true
  min?: number;          // number일 때
  max?: number;
}

interface SpecTemplate {
  params: SpecParam[];
  format: string;        // "F-CV {sq}SQ {cores}C"
}
```

### 4-3. Material on-demand 생성

```
사용자가 UTP Cat.6 4P 선택
  → Material WHERE categoryId=CBL-UTP AND specification="UTP CAT.6 4P"
  → 있으면 재사용, 없으면 자동 생성
  → Cable.materialId = 해당 Material
```

## 5. UX 설계

### 5-1. 설비 추가

기존 카테고리 선택(SERVER, NETWORK...)을 자재 카테고리 선택으로 대체한다.
추가되는 단계는 1~2개이며, 최근사용/프리셋으로 최소화한다.

```
이름 입력 → 종류 [네트워크장비 ▾] → 규격 [L2 스위치 24포트 ▾] → 추가
```

**최근 사용 / 프리셋:**
```
규격 선택:
┌─────────────────────────┐
│ ★ 최근 사용              │
│   L2 스위치 24포트       │  ← 1클릭으로 끝
│   UPS 10kVA             │
│ ─────────────────────── │
│ ▶ 직접 선택              │
│   종류: [L2 스위치   ▾]  │
│   규격: [24포트      ▾]  │
└─────────────────────────┘
```

**규격이 1가지인 경우 자동 선택:**
- PDAS → "PDAS 본체" 1개뿐 → 추가 선택 불필요

### 5-2. 케이블 연결 — 경로 클릭 방식

```
1. 케이블 도구 선택
2. 출발 클릭 (랙 → 설비 선택 팝업 / 도면 설비 직접 클릭)
3. 꺾이는 지점 클릭, 클릭, 클릭... (경유점, 격자 스냅)
4. 도착 클릭 (랙 → 설비 선택 팝업 / 도면 설비 직접 클릭)
5. 종류/규격 선택 팝업
6. 완료
```

**키보드 단축키:**
- `Enter` / 더블클릭: 경로 완료 → 종류/규격 팝업
- `Backspace`: 마지막 꺾임점 되돌리기
- `ESC`: 취소

**길이 자동 산출:**
```
경로 길이 (pathPoints 좌표 → scaleRatio 적용)
  → 1m 단위 올림 (13.1m → 14m)
  → 여유분 추가 (출발 2m + 도착 2m = +4m)
  → 총 길이 = 올림 경로 + 여유분 (14 + 4 = 18m)
```

**실시간 피드백:**
- 그리는 중 마우스 옆에 누적 길이 표시
- 완료 후 케이블 위에 규격 + 총 길이 표시
- 호버 시 상세 (경로길이, 여유분, 총길이, 출발→도착 설비명)

### 5-3. 도면 뷰 + 랙 상세 뷰

**도면 뷰 (Floor Plan View):**
- 랙은 사각형, 사용률(8/42U) 표시
- 케이블은 랙↔랙 (또는 랙↔도면설비) 사이 선으로 표시
- 같은 경로 케이블 → 번들로 묶어 표시 (UTP×3, F-CV×1)
- 배관/트레이 경로 표시

**랙 상세 뷰 (Rack Elevation View) — 랙 클릭 시:**
- U슬롯별 설비 시각화 (전면)
- 각 설비에 연결된 케이블 목록 (우측)
- 설비 추가: 이름 + 종류/규격 + 크기(U) + 위치(빈슬롯)
- 설비 클릭 → 기존 EquipmentDetailPanel (사진, 점검이력, 포트, 경로추적)
- 내진가대 토글

**케이블 연결 진입점 2가지:**
- 도면 뷰에서: 케이블 도구 → 랙 클릭 → 설비 선택 팝업 → 경로 그리기 → 도착
- 랙 상세 뷰에서: 설비 옆 [케이블 연결+] → 대상 선택 (같은 랙/다른 랙/도면 설비)

### 5-4. 설비 삭제 시 연쇄 처리

삭제를 차단하지 않고, 연쇄 영향을 보여주고 확인받는다:

```

  ☑ UTP Cat.6 4P → 랙A2:서버-1
  ☑ UTP Cat.6 4P → 랙A2:스위치-2
  ☑ 광점퍼 SC-LC 2C → OFD-1

※ 공사설계 시 위 항목이 철거 공량에 자동 포함됩니다

      [삭제]  [취소]
```

### 5-5. 부속자재 처리

부속자재는 3가지 유형으로 나눠 처리한다:

**Type A — 도면에 그리는 것:**
| 부속자재 | 도면 도구 | 자재 선택 |
|---------|----------|----------|
| 전선관 (ACC-PIPE) | 배관 도구 (경로 그리기) | 종류→규격 |
| 케이블트레이 (ACC-TRAY) | 트레이 도구 (경로 그리기) | 종류→규격 |
| 풀박스 (ACC-BOX) | 풀박스 도구 (위치 배치) | 재질→치수 |

이들은 FloorPlanElement의 elementType으로 추가:
- `conduit` (전선관), `tray` (트레이), `pullbox` (풀박스)
- 기존 line, rect, circle, door, window, text에 추가

내진가대 (ACC-SEIS)는 랙 속성으로 관리 (Rack.hasSeismicBrace)

**Type B — 자동 산출 (사용자 행위 없음):**
| 부속자재 | 트리거 | 산출 규칙 |

| 압착단자 | 전원 케이블 신설 | 케이블당 2개 |
| 광어댑터 | 광점퍼 신설 | 케이블당 2개 |
| 케이블타이 | 케이블 신설 | 2m당 1개 |
| 라벨/명판 | 설비/케이블 신설 | 각 1개 |

| 배관부속 | 전선관 신설 | 새들 1m당 2개, 인서트캡 양단 등 |

이 규칙은 ConstructionTemplateItem에 정의한다.

**Type C — Room 레벨 설정:**
| 부속자재 | 입력 위치 | 산출 |
|---------|----------|------|

## 6. 도면 상태 관리 + Diff 엔진

### 6-1. 준공 상태 (As-Built)

```
플로우:
  1. 최초 도면 작성 → 저장 → "준공 확정" → As-Built 스냅샷 생성
  2. 공사 필요 → 도면 수정 (Design 상태)
  3. "공사설계서 생성" → As-Built vs 현재 Diff → 공량 산출
  4. 공사 완료 → "준공 확정" → 새 As-Built 스냅샷
```

### 6-2. Diff 엔진이 감지하는 변경

| 대상 | 변경 유형 | actionType | 공량 영향 |
|------|----------|-----------|----------|
| 설비 | 신규 추가 | INSTALL | 설치 공수 + 부속자재 |
| 설비 | 삭제 | REMOVE | 철거 공수 + 연결 케이블 전부 철거 |
| 설비 | 위치 이동 (같은 실) | RELOCATE | 이설 공수 (< 신설+철거) |
| 설비 | 규격 변경 | INSTALL+REMOVE | 교체 공수 (구품 철거 + 신품 설치) |
| 케이블 | 신규 | WIRING | 포설 공수 + 길이 기반 자재 + 부속자재 |
| 케이블 | 삭제 | REMOVE | 철거 공수 |
| 케이블 | 경로 변경 | RELOCATE | 이설 공수 |
| 랙 | 신규 | INSTALL | 설치 공수 + 내진가대 연계 |
| 랙 | 삭제 | REMOVE | 내부 설비 전부 + 모든 케이블 철거 (연쇄) |
| 배관/트레이 | 신규/삭제 | INSTALL/REMOVE | 자재수량 + 설치/철거 공수 |

### 6-3. 이설 감지 로직

같은 ID의 설비가 Before/After에 모두 존재하지만 위치가 다른 경우:
- 같은 랙, 다른 U슬롯 → 이설
- 다른 랙 → 이설
- 랙 ↔ 도면 직접 배치 → 이설

이설은 철거+신설보다 공수가 적다 (ConstructionTemplate에 별도 정의).

## 7. 케이블 길이 산출 규칙

```
1. 경로 좌표(pathPoints)에서 각 구간 거리 계산
2. scaleRatio 적용하여 실제 거리(m)로 변환
3. 1m 단위 올림 (13.1m → 14m)
4. 여유분 추가 (기본: 출발 2m + 도착 2m = 4m)
5. 총 길이 = 올림 경로 + 여유분

예: 경로 13.1m → 올림 14m → +4m → 총 18m
```

배관/트레이도 동일 규칙 적용 (여유분 제외).

## 8. 공사설계서 산출 플로우

```
1. "공사설계서 생성" 클릭
2. As-Built 스냅샷 vs 현재 상태 Diff 계산
3. 변경 항목별 actionType 분류
4. 각 항목 → ConstructionTemplateItem 매칭
   → 공종(WorkItem) + 노무(WorkItemLabor) 산출
   → 부속자재(Type B) 자동 산출
5. 할증 규칙(SurchargeRule) 적용
6. 결과 출력:
   - 자재 수량표 (설비, 케이블, 부속자재)
   - 노무량표 (직종별 공수)
   - 변경 내역서 (Before/After 비교)
```
