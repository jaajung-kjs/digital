# ICT 디지털 트윈 시스템

## 프로그램 정의

변전소(변전소) 내 ICT 통신실의 설비, 케이블, 광경로를 웹 브라우저에서 시각적으로 관리하는 **디지털 트윈 플랫폼**입니다.

기존에 도면, 엑셀, 현장 방문에 의존하던 통신실 인프라 관리를 하나의 시스템으로 통합하여, 사무실에서 실시간으로 설비 현황을 파악하고 관리할 수 있습니다.

---

## 해결하는 문제

| 기존 방식 | 디지털 트윈 도입 후 |
|---|---|
| 종이 도면·CAD 파일로 통신실 배치 관리 | 웹 브라우저에서 2D 평면도 직접 편집 |
| 설비 정보를 엑셀로 개별 관리 | 설비·포트·케이블 정보를 한 곳에서 통합 관리 |
| 케이블 연결 현황 파악에 현장 방문 필요 | 경로 추적 기능으로 연결 경로를 즉시 시각화 |
| 변경 이력 추적 불가 | 모든 변경 사항 자동 기록 (감사 로그) |
| 유지보수 기록이 분산 | 설비별 유지보수 이력 일괄 관리 |

---

## 주요 기능

### 1. 조직·시설 계층 관리

본부 → 지사 → 변전소 → 층 → 실(통신실) 순으로 전국 시설을 체계적으로 탐색합니다.
트리 구조로 원하는 통신실까지 빠르게 이동할 수 있습니다.

### 2. 통신실 평면도 편집기

통신실의 2D 배치도를 웹에서 직접 그리고 편집합니다.

- 벽, 문, 창문 등 구조물 배치
- 랙(서버 캐비닛) 위치·회전 배치
- 설비를 랙 또는 바닥에 자유 배치
- 격자 스냅, 확대/축소, 실행취소/다시실행 지원
- 변경 사항을 일괄 저장 (변경 세트 패턴)

### 3. 설비 상세 관리

각 설비(서버, 네트워크 장비, UPS, OFD 등)의 정보를 등록하고 관리합니다.

- 분류, 모델명, 제조사, 담당자 등 기본 정보
- 포트 정보 (AC, DC, LAN, 광, 콘솔 등)
- 설비 사진 등록 (전면/후면)
- 설비별 유지보수 이력 기록 (장애, 정비, 수리)

### 4. 케이블 연결 관리

설비 간 물리적 케이블 연결을 등록하고 시각화합니다.

- 케이블 종류: AC, DC, LAN, 광(FIBER), 접지
- 평면도 위에서 연결 현황 시각 확인
- 캔버스에서 드래그로 직관적 연결 생성

### 5. 광경로(Fiber Path) 관리

변전소 간 광케이블 연결을 관리합니다.

- OFD(광분배함) 간 광경로 등록
- 포트 번호별 경로 관리
- 변전소 간 광통신 연결 구조 파악

### 6. 경로 추적 및 토폴로지 시각화

특정 케이블을 선택하면 해당 연결이 어디까지 이어지는지 전체 경로를 자동으로 추적합니다.

- 케이블 클릭 → 연결된 설비·광경로 전체 경로 하이라이트
- 경로 패널: 경유 설비와 광경로를 순서대로 표시 (예: 송광치 → 춘천-남춘천 #1 ┄ 남춘천-화천 #2 ┄ ...)
- 토폴로지 모달: 변전소 간 연결 구조를 다이어그램으로 시각화
- 링(Ring) 구조 선택으로 특정 광 링만 집중 확인

### 7. 통신실 3D 뷰

2D 평면도를 기반으로 통신실 내부를 3D로 시각화합니다.

- 랙·설비 배치를 입체적으로 확인
- 마우스로 시점 회전·확대·이동
- 현장 방문 없이도 통신실 내부 구조를 직관적으로 파악

### 8. 변경 이력 및 감사 로그

모든 데이터 변경을 자동으로 기록합니다.

- 누가, 언제, 무엇을 변경했는지 추적
- 설비 추가/수정/삭제/이동 이력
- 스냅샷 기반 변경 전후 비교

### 9. 사용자 권한 관리

- **관리자(Admin)**: 모든 데이터 편집 가능
- **뷰어(Viewer)**: 조회 전용 (읽기만 가능)

## 시스템 구성

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   웹 브라우저  │ ──▶ │  프론트엔드    │ ──▶ │   백엔드 API  │
│  (사용자 PC)  │     │  (React)     │     │  (Node.js)   │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
                                         ┌──────▼──────┐
                                         │  데이터베이스  │
                                         │ (PostgreSQL) │
                                         └─────────────┘
```

- **프론트엔드**: React 기반 웹 애플리케이션 (2D 캔버스 편집기 포함)
- **백엔드**: Node.js REST API 서버
- **데이터베이스**: PostgreSQL (모든 설비·연결·이력 데이터 저장)
- **배포**: Docker 컨테이너로 운영

---

## 기대 효과

1. **현장 방문 최소화** — 사무실에서 통신실 현황을 실시간 파악
2. **정보 일원화** — 도면, 설비 대장, 케이블 정보를 한 시스템에서 관리
3. **신속한 장애 대응** — 경로 추적으로 장애 영향 범위를 즉시 파악
4. **변경 이력 투명성** — 모든 변경이 자동 기록되어 감사·인수인계 용이
5. **업무 효율 향상** — 수작업 도면 관리를 웹 기반 편집으로 대체
